# 多主题深度研究技能

## 概述

为每个研究课题启动独立 `general-purpose` Agent 进行并行深度检索，生成精简的研究知识库。

**核心原则**：只做信息检索与归纳，不新增事实，不虚构信息，聚焦问题，精简输出。

## 输入格式

```
/multi-search @文档路径.md           # 基于文件，自动提取课题
/multi-search                        # 直接粘贴材料
/multi-search                        # 指定课题列表
项目：[项目名称]
研究课题：
1. [课题一]
2. [课题二]
```

## 处理流程

### 阶段1：分析准备

1. 读取输入材料，提取或确认研究课题清单
2. 课题拆分原则：**方向明确**（每个课题对应独特检索方向）、**避免重叠**（关键词不重复）、**聚焦问题**（每个课题解决一个具体问题）
3. 确定项目名称和输出位置

### 阶段2：输出目录检测

按优先级：`output/[项目名]/` → `./[项目名]/` → `./research/`

创建目录：`[输出目录]/03 - 🔍 深度研究/`

### 阶段3：并行深度研究

为每个课题启动独立 `general-purpose` Agent，传递：
- 项目关键信息（背景、目标、核心问题）
- 完整课题清单及各课题检索范围（避免重复）
- 具体需求背景

**去重机制**：每个 Agent 检索前声明关键词，主 Agent 审核确认无重复后再开始。发现重复时转向相关但不同的角度。

**研究要求**：
- 4-6轮深度检索，自动选择 WebSearch 或 WebFetch
- 聚焦单一核心问题，精简输出
- 包含关键来源链接（内嵌到正文），提供直接可用的结论和建议

### 阶段4：整合输出

1. 验证所有 Agent 研究结果的完整性和质量
2. 生成研究总览文档（000.研究总览.md）
3. 整合核心发现，创建文档间导航链接
4. 添加综合建议和立即行动清单

## 输出格式

详见 [references/output-templates.md](references/output-templates.md)，包含总览文档模板、详细研究文档模板、命名规范和链接规范。

## 质量标准

- **聚焦单一问题**：每个 Agent 仅解决一个核心问题
- **结构清晰**：章节标题层级分明，叙述连贯
- **链接准确**：所有来源链接内嵌到正文相应位置
- **可操作性强**：提供具体的步骤、工具、命令

## 依赖

本技能使用 Claude Code 内置工具，无需额外配置：WebSearch、WebFetch、Task（启动独立研究 Agent）。
