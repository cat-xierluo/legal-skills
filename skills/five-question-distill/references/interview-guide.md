# 无素材路径——苏格拉底式访谈问题清单

本文档对应 SKILL.md 步骤 1B。用户没有素材时，按此清单逐题访谈，从零产出五问答案。

## 访谈三条铁律

1. **一次一问**：不要一次列五个问题。
2. **给锚点不给答案**：每题配 2-3 个具体例子帮助联想，但不替用户做决定。
3. **追问直到具体**：模糊回答必追问（"帮助客户" → 哪个阶段、什么角色、什么场景）。

## 访谈顺序（按五问展开）

### 第 1 问：输入什么（先问这个，最容易答）

**引导话术**：「你平时重复做的那件事，一开始手上拿到的是什么？」

**锚点举例**：
- "比如审合同时，你拿到的是 Word 文档？还是对方发来的条款清单？"
- "比如做尽调，你拿到的是企业提供的资料包？还是公开检索结果？"

**追问**：用户答完后追问"必须有的"和"有了更好"的分别是什么。

### 第 2 问：输出什么

**引导话术**：「做完了，你交出去的是什么东西？长什么样？」

**锚点举例**：
- "是一份审查意见书？还是直接在原合同上批注？"
- "是一份正式法律意见书？还是给客户的一封邮件？"

**追问**：追问格式（字数、章节、是否要正式抬头）和质量标准（什么算"做得好"）。

### 第 3 问：处理逻辑

**引导话术**：「从拿到东西到交出去，你脑子里走了哪几步？」

**锚点举例**：
- "审合同你是不是先看主体，再看标的，然后逐条扫风险？"
- "写律师函你是不是先理事实，再找法条，最后定调子？"

**追问**：问决策点——"哪些地方你会根据情况走不同路？"（这是 workflow 的关键）。

### 第 4 问：向谁交付（本书补的关键问）

**引导话术**：「这个东西最后给谁看？他什么情况下会用到？」

**锚点举例**：
- "给客户本人看？还是给对方律师看？还是给法官看？"
- "客户是在签约前看？还是出事后看？"

**追问三连**：
1. 角色：终端用户是谁（律师同行 / 客户 / 法官 / 对方）
2. 场景：什么触发条件下用（签约前 / 纠纷发生后 / 日常合规）
3. 语气：要正式威慑？还是要平和可读？

这一问决定整个 Skill 的语气基调，**不能跳过**。

### 第 5 问：需要哪些知识支撑（本书补的关键问）

**引导话术**：「你做这件事的时候，脑子里或手边靠的是什么？」

**锚点举例**：
- "靠某几部法条？《民法典》哪几编？《公司法》哪几条？"
- "靠你自己积累的风险清单？还是律所内部的范本库？"
- "靠某位资深律师传授的口诀或经验？"

**追问**：
1. 法律依据：哪几部法律、哪几条、哪个司法解释
2. 范本来源：自己积累 / 律所库 / 公开模板
3. 风险清单：有没有现成的、谁维护
4. 风格偏好：跟谁的风格、有没有不允许的表述

这一问决定 Skill 是"空壳模板"还是"有肉的真本事"，**不能跳过**。

## 访谈节奏控制

- 五问不必一轮问完。如果用户答到一半卡住，先消化已有信息，下次继续。
- 每问答完，**复述给用户确认**："我理解你的意思是 XX，对吗？"——避免理解偏差累积到最后返工。
- 全部答完后，进入 SKILL.md 步骤 3 产出标准 prompt。

## 访谈产出物

访谈结束后应有：

- 五问每问的具体答案（不是模糊描述）
- 一份"用户原话 → 标准化表述"的对照（便于后续 prompt 写作）
- 一份"待确认清单"（访谈中用户说"我再想想"的项）
