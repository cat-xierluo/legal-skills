# 发票字段识别指南

各类电子发票/票据的文本结构与必抓字段。提取文本后，按本指南定位字段。

## 通用解析策略

1. 先判类型：文本含"铁路电子客票"/"电子发票（普通发票）"/"航空运输电子客票"等标题词
2. 按类型套对应字段表
3. 全角数字/字符（０１２３…Ｇ￥）统一转半角再比对
4. 金额以"价税合计（小写）"为准；铁路客票以"票价"为准
5. 日期统一转 YYYY-MM-DD 输出

## 铁路电子客票

文本特征：标题"电子发票（铁路电子客票）"。

| 字段 | 文本定位 |
|------|----------|
| 发站 / 到站 | 站名成对出现，含拼音（"苏州北站 Suzhoubei ... 石家庄站 Shijiazhuang"） |
| 车次 | "G1220" 等，位于发到站之间 |
| 乘车日期 | "2026年06月15日" |
| 开车时间 | "14:31开" |
| 座位 | "04车15F号" |
| 席别 | "二等座" 等 |
| 票价 | "票价:￥548.00" |
| 乘车人 | 身份证号后的人名（示例"张某"） |
| 电子客票号 | "电子客票号:..." |
| 发票号码 | "发票号码:..." |
| 购买方 | "购买方名称:..." |
| 开票日期 | "开票日期:..." |

注意：乘车日期（业务发生日）≠ 开票日期（常滞后）。归档命名与清单"日期"列一律用**乘车日期**。

## 增值税电子普通发票（住宿/服务）

文本特征：标题"电子发票（普通发票）"，含"购 名称""销 名称"两栏。

| 字段 | 文本定位 |
|------|----------|
| 购买方名称 | "购 名称：..." |
| 购买方税号 | 购买方栏下"统一社会信用代码/纳税人识别号" |
| 销售方名称 | "销 名称：..." |
| 项目名称 | 明细表"*分类*服务名"，如"*生产生活服务*住宿服务" |
| 数量 / 单价 / 金额 / 税率 / 税额 | 明细表列 |
| 价税合计 | "价税合计（大写）...（小写）¥xxx" |
| 发票号码 | "发票号码：..." |
| 开票日期 | "开票日期：..." |

住宿类摘要建议含酒店名 + 晚数（如"石家庄XX酒店 2 晚"）。金额取价税合计（小写）。
