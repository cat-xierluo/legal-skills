# 合同型场景评测维度清单

适用范围：合同初审、合同审查、合同起草、合同风险表、补充协议起草等以"合同文本"为处理对象的 Skill 输出。

## 一、维度清单

### 维度 1：法律准确性

检查问题：

- 是否正确理解合同所涉规则、法律关系和基本概念（如违约责任、解除权、不安抗辩、违约金调整）
- 是否把不确定结论写成确定结论（Hard Fail 邻接项，应回到 skill-lint）
- 对本场景核心权益（如采购方在服务合同中的服务稳定性、付款方在买卖合同中的验收与付款挂钩）的判断是否准确

低分表现：法条或概念错误；把商业谈判点写成法律风险；把事实缺口写成法律结论。

高分表现：规则表述准确；结论边界清楚；区分法律风险、商业谈判点、事实缺口、需人工确认事项。

### 维度 2：事实叙述与条款定位

检查问题：

- 是否忠实基于已提供合同材料（不虚构交易背景）
- 重点风险是否尽量标注条款编号或条款位置；无编号时是否说明"合同未标明条款编号"
- 是否遗漏关键条款（付款 / 验收 / 违约 / 解除 / 知识产权 / 保密 / 争议解决 / 适用法律）

低分表现：虚构背景；遗漏关键事实；引用条款但说不清在哪。

高分表现：能对应到具体条款位置；事实缺口明确标注。

### 维度 3：风险分级与争议焦点把握

检查问题：

- 是否区分高 / 中 / 低风险，而不是平均罗列
- 是否抓住对该方（采购方 / 供应商 / 委托方）实际影响决策的高影响条款
- 是否识别单方变更、单方终止、单方暂停、自动续期、责任上限、争议解决条款等典型高影响条款

低分表现：风险平均罗列；主次不分；把低影响条款列为重点风险。

高分表现：识别关键冲突；按对本方影响排序。

### 维度 4：逻辑说服力

检查问题：

- 是否从条款 → 风险 → 建议逐步展开
- 风险与建议是否一一对应
- 是否有"只有结论没有理由"的孤立条目

低分表现：只有结论；前后矛盾；建议与风险脱节。

高分表现：链条清楚；可被业务同事或谈判对手理解。

### 维度 5：实务可用性（修改方向 + 需确认事项）

检查问题：

- 每项风险是否有具体修改方向（而不是"建议进一步审查"这类空话）
- 是否列出需业务 / 客户确认的事项（如商业意图、可接受的风险敞口）
- 输出能否直接进入红线修改、谈判或内部讨论

低分表现：建议空泛；无法修改、沟通或分派。

高分表现：行动项明确；可复核、可沟通、可修改。

### 维度 6：交付物格式完整度

检查问题：

- 是否符合本场景交付物结构（风险表 / 初审意见 / 修改建议清单）
- 字段是否稳定（风险描述 / 条款位置 / 风险等级 / 修改方向 / 需确认事项）
- 层级是否清楚、读者能否快速定位

低分表现：字段缺失；层级混乱；读者难定位。

高分表现：字段稳定；层级清楚；适配交付对象（内部业务 / 外部客户）。

## 二、合同型场景 taste 项（必须显式核查，不并入维度）

以下是经验律师在合同审查中一眼就会觉得"不对"的东西。它们不直接对应上面六个维度，但任何一项命中都会让整份输出失去可信度。评测时必须逐项核查并单列。

- taste-1：是否区分"法律风险"和"商业谈判点"。把"价格偏高"写成法律风险，是典型越界。
- taste-2：是否区分"风险"和"事实缺口"。材料里没说的事情，应标"需确认"，不能编。
- taste-3：是否站在本方立场。采购方视角的合同审查，把"保护供应商"列为重点风险，是立场错位。
- taste-4：是否提示材料缺失会影响判断（如缺主合同、缺附件、缺报价单）。
- taste-5：是否避免过度承诺（如"该合同无任何法律风险"这类绝对化表述）。
- taste-6：是否避免把草稿写成正式法律意见（应明确"初审意见，非正式法律意见"）。

## 三、与 ch07 第六节六维度的对应关系

本清单的六个维度与 ch07 第六节六维度评价标准（法律准确性 / 事实叙述与证据运用 / 逻辑说服力 / 争议焦点把握 / 实务可用性 / 文书格式完整度）一一对应，但做了合同场景专属化：

- "事实叙述与证据运用"在本场景具体化为"事实叙述与条款定位"
- "争议焦点把握"在本场景具体化为"风险分级与争议焦点把握"
- 其余维度保留通用含义
