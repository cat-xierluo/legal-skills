# 六维度评价标准的来源与降级说明

## 一、来源

六维度评价标准来自本书（法律 skill 书籍）第七章第六节"第四步：怎样验证初版 Skill 是否可用"的"四、六维度评价标准：检查法律输出质量"小节。

原文给出的六个维度：

| 维度 | 检查问题 |
|------|---------|
| 法律准确性 | 是否正确理解规则、法律关系和基本概念 |
| 事实叙述与证据运用 | 是否忠实基于已提供材料 |
| 逻辑说服力 | 是否有清楚推理链条 |
| 争议焦点把握 | 是否抓住实际影响决策的问题 |
| 实务可用性 | 输出能否进入下一步工作 |
| 文书格式完整度 | 是否符合交付物结构 |

原文同时给了一个 1-5 分初评机制，并强调"这张表的作用是帮你定位改进点，而不是让 AI 自己给自己打高分"。

## 二、为什么本技能不把它作为通用基准

原文章节语境是**合同初审 Skill**（供应商服务合同初审）。在这个语境下，六维度基本适用。

但 ch07 第六节 #91 联动调整后，本书第七章第六节的定位从"给所有法律 Skill 一套通用基准"调整为：

- 合同型场景：六维度基本适用。
- 诉讼型、合规型等其他场景：六维度覆盖不全，需各自场景专属维度。

因此本技能的处理是：

- **六维度作为合同型场景的参考默认起点**（见 `references/scenarios/contract-scenario-rubric.md`，六维度一一对应，但做了合同专属化）。
- **诉讼型场景**：请求权基础、诉讼请求明确性、证据组织、抗辩预判等维度不在六维度覆盖范围内（见 `references/scenarios/litigation-scenario-rubric.md`）。
- **合规型场景**：监管依据精确度、风险等级判定、整改路径可执行性等维度不在六维度覆盖范围内（见 `references/scenarios/compliance-scenario-rubric.md`）。

## 三、不输出通用总分

本技能**不输出**以下内容：

- 一个适用于所有法律 Skill 的"六维度通用基准"和单一总分。
- 跨场景的"平均分"或"加权总分"。
- 把不同场景 Skill 排进同一张榜单的横向比较。

理由：法律场景之间的差异远大于共性。强行套基准，评测本身会变成另一个看起来严谨但没用的产物。

## 四、与作者既有文书质量评测尝试的关系

作者既有尝试（六维度评价标准）是本技能的**起点之一**，但不是全部。本技能在此基础上：

- 保留六维度作为合同型场景的参考起点。
- 补充诉讼型、合规型的专属维度。
- 在每个场景维度清单末尾单列"场景 taste 项"，把经验律师一眼会觉得不对的东西显式化，不让通用维度稀释它。
- 明确与 skill-lint 的规整性层划界，避免两层评测混在一起。

## 五、业界参考但不照搬

本技能参考了两类业界思路（v0.2.0 依据 DR-5 前沿调研更新表述）：

- **Anthropic eval 方法论 + Claude skill creator 的 eval 验证思路**（test cases / expected behavior / assertion grading / benchmark / blind A / description tuning）：借鉴"用熟悉材料做基准"的思路。但 v0.2.0 起**不再使用"黄金答案"这一术语**——DR-5 明确标注该术语未在 Anthropic 官方公开材料核验到（详见 `references/eval-methodology.md` 第二节）。改用"参考解三档口径"（reference-rich / reference-light / reference-free）：可枚举任务写 reference solution 用 deterministic + LLM rubric 对照；多解任务写 ideal response 加 material omission/incorrectness/hallucination 规则；强主观任务用 LeMAJ 式 reference-free 分解 + pairwise 专家偏好。法律输出的人工 taste 判断无法被自动比对替代。
- **作者既有六维度评价标准**：降级为合同型场景的参考起点。

两者都不照搬，因为法律场景的多样性决定了任何单一基准都不通用。
