# 与 skill-lint 的对接协议

skill-lint 负责"规整性层"（目录结构、frontmatter、引用一致性、触发词边界、安全、发布版本、Hard Fail、业务流深度五层）。skill-evaluation 负责"输出合理性层"（在某个法律场景里输出内容对不对）。

本文件规定两者如何对接。

## 一、前置：本技能读取 skill-lint 的哪些结论

启动内容评测前，本技能应读取或调用 skill-lint 的以下结论：

### 1. Hard Fail 结论

skill-lint 的 Hard Fail 条件（来自 `references/business-flow-rubric.md`）：

- 编造关键事实、证据、时间、主体或依据
- 未标注信息不足，却给出确定性专业结论
- 将辅助分析、初筛或草稿表述为正式意见
- 引用明显错误或与结论不相关的依据
- 在高风险场景下没有提示人工复核或专业人员介入
- 未按 Skill 自身规则输出必要免责声明、风险提示或升级建议
- 输出会泄露本应脱敏的个人、企业或案件敏感信息
- 公开示例、配置模板或参考文档中出现真实人名、客户名、案件项目、案号或可反查组合信息
- 文档没有可执行流程，只是概念、原则或口号堆叠

处理规则：

- 任一 Hard Fail 命中 → 本技能**先提示用户回 skill-lint 修 Hard Fail**，再启动内容评测。
- 用户坚持要做内容评测时，本技能可执行，但在报告"总体结论"里必须标 ❌ 回退到 Hard Fail，并把该 Hard Fail 列为最高优先级修补项。
- 不在本技能里重新判定 Hard Fail——那是 skill-lint 的职责。

### 2. 可评估性基础五项

skill-lint 的"可评估性审查"五项（来自 SKILL.md 第六节）：

- 是否声明评估范围
- 是否声明 Hard Fail
- 是否提供 benchmark case 或样例
- 是否提供输出验收标准
- 是否区分静态检查与动态评估

处理规则：

- 缺失任一项，在本技能报告"前置——结构是否可评测"段记为"结构性缺口"。
- 不阻塞本次内容评测，但报告须明确"本次评测基于不完整结构"。
- 不替 skill-lint 出正式质量意见报告——本技能只引用其结论。

### 3. 业务流深度五层

skill-lint 的 Trigger / Intake / Reasoning / Output / Safety 五层判定的是**文档是否承载了流程**。

处理规则：

- 本技能**不在文档层面重复判定**这五层。
- 但本技能的"输出合理性"评测结果，会反向印证这五层是否真的有效：例如 Output 层在文档里声明了输出验收标准，但本技能跑测试材料时发现输出根本不达标，则在本技能报告里提示"文档层 Output 声明的验收标准与实际输出存在落差，建议回 skill-lint 复核 Output 层"。

## 二、不重叠的边界

| 评测对象 | skill-lint | skill-evaluation |
|---------|-----------|------------------|
| 目录结构、文件可达性 | ✅ | ❌ |
| frontmatter 字段合规 | ✅ | ❌ |
| 触发词边界（name/description） | ✅ | ❌ |
| 安全评估（凭证、危险执行、数据外传、提示注入） | ✅ | ❌ |
| 发布治理（LICENSE / CHANGELOG / version） | ✅ | ❌ |
| Hard Fail 判定 | ✅（权威） | 引用，不重判 |
| 业务流深度五层（文档层） | ✅ | 反向印证，不重判 |
| 可评估性基础五项（文档层） | ✅（权威） | 引用为前置 |
| 在某法律场景的输出内容质量 | ❌ | ✅ |
| 按场景的评测维度集 | ❌（只给 business-flow-rubric 通用五层） | ✅（contract / litigation / compliance 各一套） |
| 场景 taste 项核查 | ❌ | ✅ |
| 定位"要补 SKILL.md 哪句话" | ❌（不针对具体输出） | ✅ |

## 三、skill-lint 的 business-flow-rubric 与本技能场景维度的关系

skill-lint 的 `business-flow-rubric.md` 明确写道（第 1 节）：

> 不要把本判则用于评判具体任务答案是否正确。具体答案质量应进入 benchmark eval 或人工复核。

本技能正是填补这一空白：

- business-flow-rubric 评"流程是否被承载"——文档层。
- 本技能场景维度清单评"流程跑出来的结果对不对"——内容层。

两者互补，不冲突，不重叠。

## 四、当 skill-lint 不可用时

如果用户环境没装 skill-lint，或被评测 Skill 还没到能过 skill-lint 的阶段，本技能按降级模式工作：

- 跳过 skill-lint 前置读取，直接进入内容评测。
- 在报告"前置——结构是否可评测"段标"未对接 skill-lint，本次评测为降级模式"。
- 提示用户：完整评测应包括 skill-lint 的规整性层 + 本技能的输出合理性层。
