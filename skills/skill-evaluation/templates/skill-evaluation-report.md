## 五-bis、四校准动作记录（v0.2.0 新增，仅当本次启用 LLM judge 时填写）

> 依据 SKILL.md「LLM-as-judge 四校准」（别名 四-C）与 `references/eval-methodology.md` 第三节。纯人工评测则在本段标"本次未启用 LLM judge"并跳过。

| 校准动作 | 本次是否启用 | 配置 / 记录 |
|---------|------------|------------|
| 锚点样本（anchor sample） | ✅ / ❌ | 每场景 3-5 例好/中/差锚点是否齐备 |
| 顺序翻转（pairwise 防位置偏差） | ✅ / ❌ / N/A | pairwise 样本是否做了左右翻转；翻转后 verdict 翻转的样本编号 |
| 多 judge 共识（PoLL 异质 panel） | ✅ / ❌ | 强 judge + 便宜 judge 组合；低置信度样本升级人工编号 |
| 证据锚定（evidence-anchored） | ✅ / ❌ | judge 是否给出所依据法源/条款/片段（不接受裸分） |

**长文 judge 单独建模**（诉讼型 / 合规型长文必填）：

- 本次输出是否属长文（答辩状 / 起草状 / 合规 memo 等）：✅ / ❌
- 若是长文，judge panel 是否做了更保守配置（更多锚点样本 / 更窄 rubric / 更多人工抽检）：✅ / ❌

**pairwise vs 绝对评分协议选择**：

- 本次"是否完整/是否误导/是否虚构/是否遗漏"类判断 → 用绝对评分：✅
- 本次"哪份更像能发出去/哪份更有说服力"类偏好 → 用 pairwise + 顺序翻转：✅ / 未涉及
