# 裁判 Prompt 模板

> 本模板用于 Step 2：裁判模型对 N 份候选草稿打分。直接复制下方模板，填入 N 份草稿后喂给裁判模型。

## 使用说明

- 裁判模型必须**不在候选清单里**（避免自评偏差）。
- N 份草稿要**同时**给裁判，让它横向比较，而不是逐份独立评。
- 裁判输出必须严格按模板的 Markdown 结构，方便后续聚合排名。

````markdown

## 评价标准（必须严格按这两套既有规范，不另起炉灶）

### 标准一：本书《书稿口吻 Review》七类口吻门禁
对每份草稿逐类扫描命中处：
1. **调研报告口吻**：公开资料显示 / 官方文档描述为 / 资料不足以详写 / 从公开信息看 / 进入公众视野 等。
2. **写作取舍口吻**：适合进入本章 / 本章可以保留 / 值得单独讲 / 更稳妥的写法 / 正文主线应当 等。
3. **模板化对比句和宣言句**：不是…而是 / 不在于…在于 / 看似…实则 / 这不是…这是 / 从…走向 等（连续出现即记）。
4. **空泛意义拔高**：深刻改变 / 彰显 / 赋能 / 格局演变 / 未来可期 / 方向已经明确 等。
5. **工整排比和重复疑问**：连续三个"能不能 / 如何 / 怎样 / 既要…又要…还要"。
6. **框架提示句、冒号定义句和碎句感**：可以从…角度理解 / 本质上是 / 核心在于 / A：B 冒号定义 / 连续短句收束。
7. **高频模糊词**：越来越 / 其实 / 往往 / 通常 / 很多时候 / 真正 / 更像 / 不只是 / 很容易 / 更重要的是（同小节反复即记）。

### 标准二：de-ai-polish 检测方法
- **频次红线**：以下词全文超过 2 次即记为 AI 程式化：越来越 / 越…越 / 其实 / 往往 / 通常 / 很多时候 / 经常 / 常常 / 更重要的是 / 更麻烦的是 / 看起来 / 看来 / 真正 / 不只 / 不只是 / 更像 / 很容易 / 更容易。
- **向上溯源**：未列出的表述，判断是否属于已知模式家族变体（对比句式 → 递进结构 → 程式化转折 → 工整排比 → 程度副词堆叠 → 否定宣言 → 分离式对比 → 同义循环）。
- **结构机械感**：三并列同虚词引导 / 程式化段首引导词（更麻烦的是 / 说到这里）/ 程度副词 + 抽象名词固定搭配。

## 评分维度（每项 1-5 分，5 分最好）

| 维度 | 权重 | 评分依据 |
|------|------|---------|
| 口吻干净度 | 30% | 七类口吻门禁命中数；命中越少分越高 |
| 句式自然度 | 25% | 对比句 / 排比 / 碎句感 / 程式化引导词 |
| 词汇准确度 | 20% | 法律术语准确 + 高频模糊词控制 |
| 信息密度 | 15% | 是否有具体事实 / 场景 / 操作，而非空泛拔高 |
| 法律边界感 | 10% | 风险边界 / 操作边界是否清楚，不为自然度牺牲准确性 |

## 输出格式（必须严格按此结构）

### 1. 逐份扫描结果

对每份草稿，列出：

```
#### 草稿 <X>（<模型名>）
- 七类口吻命中：
  - 第 1 类（调研报告口吻）：<命中条数> 处。例：<引用 1-2 句原文 + 位置>
  - 第 2 类（写作取舍口吻）：<命中条数> 处。例：...
  - ...（七类逐类列）
- 频次红线词：<词 → 出现次数>，超 2 次的标红
- 结构机械感：<列出>
- 五维评分：口吻 X / 句式 X / 词汇 X / 信息 X / 边界 X
- 加权总分：X.X / 5
- 最致命问题（2-3 条）：<具体到段落和句子>
- 独特优势（1-2 条）：<具体到段落和句子>
```

### 2. 横向排名

| 排名 | 模型 | 总分 | 一句话评价 |
|------|------|------|----------|
| 1 | <模型> | X.X | <10-20 字> |
| 2 | <模型> | X.X | ... |

### 3. 选优建议

- **首选定稿模型**：<模型>
- **首选理由**：<3-5 句，结合致命问题和独特优势>
- **回退策略**（如果首选不稳）：<如"用第二名跑初稿 + de-ai-polish 单点修弱项"，或"前两名分差 ≤ 0.5，需第二轮 tiebreak">
- **一句话结论**：<这类写作任务（<任务类型>），<模型> 最稳>

## 评价纪律

- 不得使用任何"不是 X 而是 Y""看似 X 实则 Y"这类模板化对比句写评价（你正在评的就是这种句式）。
- 不得用空泛拔高（深刻 / 彰显 / 赋能）。
- 命中处必须引用原文 + 标位置，不要只说"有 AI 味"。
- 法律术语和事实准确性是底线：如果某份草稿有事实错误或法律边界不清，即使文笔自然也要在"法律边界感"扣分并单独标出。
````

## 裁判模型选择建议

| 候选模型家族 | 建议裁判 |
|------------|---------|
| 全是国产模型（GLM / DeepSeek / Qwen） | GPT-4 / Claude Opus |
| 含 Claude 家族 | GPT-4 / Gemini 旗舰 |
| 含 GPT 家族 | Claude Opus / Gemini 旗舰 |
| 全是同一公司不同型号 | 跨公司强模型（避免同公司自评偏差） |

> 裁判的核心要求：**与所有候选不同家族** + **本身去 AI 化能力强**（否则裁判自己写出来的评价也是 AI 味）。
