# 去 AI 化评分维度（对接既有规范）

> 本文件是裁判打分的维度定义。**所有维度的判定依据都来自既有规范**，本文件不另起炉灶，只把分散在 BOOK-TONE-REVIEW 和 de-ai-polish 里的标准聚合成一张可打分的表。

## 五维评分表

每项 1-5 分（5 分最好），按权重加权得总分。

| 维度 | 权重 | 5 分（优秀） | 1 分（很差） | 判定依据来源 |
|------|------|------------|------------|------------|
| **口吻干净度** | 30% | 七类口吻门禁零命中 | 命中 3 类以上 | BOOK-TONE-REVIEW §三（七类门禁） |
| **句式自然度** | 25% | 无对比句 / 无工整排比 / 无碎句感 | 连续对比句 + 三并列 + 段末短句拔高 | BOOK-TONE-REVIEW §三.3 / §三.5 / §三.6 + de-ai-polish §一.1 / §一.6 |
| **词汇准确度** | 20% | 法律术语准确 + 高频模糊词 ≤ 1 次 | 术语口语化 + 频次红线词 > 2 次 | de-ai-polish §总则.2（频次红线）+ §一.4（口语化） |
| **信息密度** | 15% | 每段有具体事实 / 场景 / 操作 | 全是空泛拔高和意义陈述 | BOOK-TONE-REVIEW §三.4（空泛意义拔高）+ §四（允许保留的内容） |
| **法律边界感** | 10% | 风险边界 / 操作边界清楚 | 有事实错误或边界不清 | BOOK-TONE-REVIEW §七（最低通过标准） |

## 各维度详细判定

### 维度一：口吻干净度（权重 30%）

**对接**：BOOK-TONE-REVIEW §三 的七类口吻门禁。

逐类扫描，记每类命中条数：

| 类别 | 代表性问题词 | 计分方式 |
|------|------------|---------|
| 1. 调研报告口吻 | 公开资料显示 / 资料不足以详写 / 进入公众视野 | 命中 0 处 → 5 分；1 处 → 4 分；2 处 → 3 分；3+ 处 → 1-2 分 |
| 2. 写作取舍口吻 | 适合进入本章 / 值得单独讲 / 正文主线应当 | 同上 |
| 3. 模板化对比句 | 不是…而是 / 看似…实则 / 从…走向 | 同上（连续出现加倍扣） |
| 4. 空泛意义拔高 | 深刻改变 / 彰显 / 赋能 / 未来可期 | 同上 |
| 5. 工整排比和重复疑问 | 连续三个"能不能 / 如何 / 怎样" | 同上 |
| 6. 框架提示句 / 冒号定义 / 碎句感 | 可以从…角度理解 / 本质上是 / A：B | 同上 |
| 7. 高频模糊词 | 越来越 / 其实 / 往往 / 真正 / 更像 | 同小节反复即记；全文超 2 次加倍扣 |

> 七类加权平均得本维度分数。

### 维度二：句式自然度（权重 25%）

**对接**：BOOK-TONE-REVIEW §三.3 / §三.5 / §三.6 + de-ai-polish §一.1 / §一.6。

重点看四种结构：

1. **对比句式家族**（de-ai-polish §一.1）：不是…而是 / 并非…而是 / 看似…实则 / 这不是…这是 / 否定宣言 / 肯定宣言。出现 1 处扣 0.5 分，连续出现直接降到 2 分以下。
2. **工整排比**（BOOK-TONE-REVIEW §三.5 + de-ai-polish §一.6）：三个以上相同结构并列（既要…又要…还要 / 三个"讲清楚" / 三个疑问词引导）。
3. **碎句感**（BOOK-TONE-REVIEW §三.6）：连续短句收束（这很关键。也很现实。更值得注意。）。
4. **程式化引导词**（de-ai-polish §一.7 / §一.9）：段首"更麻烦的是""说到这里""表面上看"。

### 维度三：词汇准确度（权重 20%）

**对接**：de-ai-polish §总则.2（频次红线）+ §一.4（口语化）。

两层判定：

1. **法律术语准确性**：Agent / Skill / MCP / CLI / API / 请求权基础 / 涵摄推理 等术语是否用对、用准。术语口语化（如把"请求权基础"说成"请求的根基"）直接扣到 2 分以下。
2. **频次红线词控制**：de-ai-polish §总则.2 列的词（越来越 / 其实 / 往往 / 通常 / 很多时候 / 经常 / 常常 / 更重要的是 / 更麻烦的是 / 看起来 / 看来 / 真正 / 不只 / 不只是 / 更像 / 很容易 / 更容易），全文超过 2 次即记。每超 1 次扣 0.3 分。

### 维度四：信息密度（权重 15%）

**对接**：BOOK-TONE-REVIEW §三.4（空泛意义拔高）+ §四（允许保留的内容）。

判断每段是否提供以下之一：
- 具体事实（时间、版本号、配置项、案例细节）
- 具体场景（谁在什么任务中做了什么）
- 具体操作（步骤、命令、配置）
- 清晰的边界（风险、限制、适用条件）

全是空泛意义陈述（深刻改变 / 彰显 / 为…做出贡献 / 具有重要意义）的段落，整段计 0 分。BOOK-TONE-REVIEW §四 明确允许保留的内容（术语、必要连接词、步骤、案例框、风险边界）不计扣分。

### 维度五：法律边界感（权重 10%）

**对接**：BOOK-TONE-REVIEW §七（最低通过标准）最后一项。

底线判定：
- 有事实错误（法律条文引用错、案例事实错、技术细节错）→ 直接 1 分，并在报告中单独标出。
- 风险边界不清（如建议读者做某操作但没说权限 / 数据出境 / 复核边界）→ 扣到 3 分以下。
- 为追求自然度牺牲准确性（如把"必须人工复核"软化为"可以考虑复核"）→ 扣到 3 分以下。

> 这一维度的权重不高（10%），但它是**否决项**：如果某份草稿法律边界感 = 1 分（有事实错误或边界严重不清），即使其他四维都 5 分，也不能选为定稿模型。

## 加权总分计算

```
总分 = 口吻干净度 × 0.30 + 句式自然度 × 0.25 + 词汇准确度 × 0.20 + 信息密度 × 0.15 + 法律边界感 × 0.10
```

总分 5 分制，保留 1 位小数。

## 排名与选优阈值

| 总分区间 | 含义 | 处理 |
|---------|------|------|
| ≥ 4.5 | 优秀，可直接定稿 | 首选 |
| 4.0 - 4.4 | 良好，定稿前走一次 de-ai-polish 单点修 | 首选或备选 |
| 3.5 - 3.9 | 合格但有明显 AI 味 | 备选，需较多 polish |
| 3.0 - 3.4 | 勉强 | 不建议定稿，除非没有更好选项 |
| < 3.0 | 不合格 | 直接淘汰 |

**并列处理**：前两名总分差 ≤ 0.5 时，建议做第二轮 tiebreak（各跑一段相同样量的不同段落，重新评分）。
