# 起诉状与证据分析文档

本文档定义了起诉状与证据的完整处理流程，这是诉讼分析的起点，用于案件初期的快速评估和策略制定。

---

## 识别特征

### 起诉状特征

- **文书类型**：民事起诉状、行政起诉状等
- **核心结构**：原告信息、被告信息、诉讼请求、事实与理由、证据列表
- **篇幅**：通常 5-20 页
- **目的**：启动诉讼程序，明确诉讼请求

### 证据材料特征

- **证据类型**：书证、物证、视听资料、证人证言等
- **组织形式**：证据目录、证据清单、举证册
- **证明目的**：支撑起诉状中的各项主张

---

## 抽取规范

起诉状与证据分析是诉讼分析的起点，用于快速评估案件基础。

### 抽取原则

- **快速评估**：快速识别案件核心要素
- **风险识别**：发现证据缺口和法律风险
- **策略导向**：为后续策略制定提供基础

### 基础信息抽取

| 字段 | 提取规则 | 说明 |
|------|---------|------|
| 原告 | 提取原告名称及基本信息 | 包含身份信息 |
| 被告 | 提取被告名称及基本信息 | 包含身份信息 |
| 案由 | 从起诉状中提取案件性质 | 如：合同纠纷、侵权纠纷等 |
| 诉讼标的 | 涉案金额或具体请求 | 用于风险评估 |
| 管辖法院 | 受理法院名称 | 判断地域管辖 |

### 诉讼请求抽取

**诉讼请求清单**：
1. 请求编号（按顺序 1、2、3...）
2. 请求类型（给付之诉/确认之诉/形成之诉）
3. 请求内容（具体诉求）
4. 金额/数量（量化指标）
5. 法律依据（支持该请求的法条）

**关键要素**：
- 请求是否明确、具体
- 请求是否有法律依据
- 请求之间是否存在逻辑关系
- 是否存在预备性请求

### 事实与理由抽取

**事实主张**：
- 时间线（按时间梳理关键事件）
- 法律关系（合同关系、侵权关系等）
- 违约/侵权行为（被告的违法行为）
- 损害后果（原告遭受的损失）

**法律理由**：
- 法律关系性质认定
- 违约/侵权责任分析
- 损害赔偿计算依据
- 相关法律条文引用

### 证据清单抽取

**证据列表**：
- 证据编号
- 证据名称
- 证据类型（书证/物证/视听资料等）
- 证明目的（用于证明什么事实）
- 证明力评估（强/中/弱）

**证据链分析**：
- 证据是否完整（能否形成完整证据链）
- 证据缺口（缺少哪些关键证据）
- 证据矛盾（证据之间是否存在矛盾）
- 证明力评估（整体证明力强弱）

---

## 处理流程

### 起诉状与证据分析流程

1. **基础信息提取**：提取原告、被告、案由、诉讼标的等基本信息
2. **诉讼请求分析**：梳理诉讼请求，评估请求的合理性和可执行性
3. **事实主张梳理**：按时间线梳理事实，识别关键事实节点
4. **证据清单整理**：整理证据清单，评估证据完整性和证明力
5. **风险识别**：识别法律风险和证据缺口
6. **策略建议**：提供初步的诉讼策略建议

## 输出侧重

### 内部版

- 📋 案件核心要素速览
- 诉讼请求分析（逐项评估）
- 事实主张梳理（时间线）
- 证据清单与证明力评估
- 证据缺口分析
- 法律风险识别
- 初步策略建议

### 客户版

- 案件基本情况说明
- 诉讼请求通俗解释
- 证据状况评估
- 预期风险提示
- 建议收集的补充证据

---

## 抽取质量检查

### 完整性检查

- [ ] 基础信息：原告、被告、案由、诉讼标的齐全
- [ ] 诉讼请求：全部诉讼请求已提取
- [ ] 事实主张：关键事实节点完整
- [ ] 证据清单：主要证据已列明

### 准确性检查

- [ ] 原告/被告信息准确
- [ ] 诉讼标的金额计算正确
- [ ] 证据与证明目的对应

### 逻辑性检查

- [ ] 诉讼请求与事实主张对应
- [ ] 证据与诉讼请求匹配
- [ ] 时间线逻辑连贯
