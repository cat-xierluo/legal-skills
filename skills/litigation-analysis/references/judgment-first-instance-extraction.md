# 一审判决书处理文档

本文档定义了一审判决书的完整处理流程，包括识别特征、抽取规范、处理流程和输出侧重。

---

## 识别特征

### 一审判决书特征

- **案号格式**："(20XX)X民初XXXX号"
- **章节结构**：原告诉称、被告辩称、本院查明、本院认为、判决如下
- **篇幅**：通常 100-500 行

---

## 抽取规范

一审判决书是 litigation-analysis 的核心处理对象之一，采用结构化深度抽取。

### 抽取原则

- **内容多**：通常 100-500 行，信息密集
- **结构规范**：有固定的章节结构和格式
- **法律逻辑严密**：事实认定、法律适用、判决结果层层递进

抽取时需要：完整提取、保持逻辑、结构化呈现

### 基础信息抽取

| 字段 | 提取规则 | 说明 |
|------|---------|------|
| 案号 | 正则匹配"(20XX)X民初XXXX号" | 一审用"初" |
| 法院 | 从法院名称中提取完整名称 | 包含省市区信息 |
| 案由 | 从文书标题或内容中提取 | 识别案件性质 |
| 原告 | 提取原告名称及基本信息 | 包含身份信息 |
| 被告 | 提取被告名称及基本信息 | 包含统一社会信用代码 |
| 审理程序 | 识别普通程序/简易程序 | 影响审限和上诉期 |
| 开庭时间 | 从审理经过中提取 | 用于时间线梳理 |
| 裁判日期 | 从文书末尾提取 | 计算上诉期的基准 |

### 事实认定抽取

**原告诉称**：
- 主张事实（按时间或逻辑顺序梳理）
- 诉讼请求（分项列出全部诉讼请求）
- 提交证据（列出原告提交的关键证据）

**被告辩称**：
- 抗辩理由（包括事实抗辩和法律抗辩）
- 提交证据（列出被告提交的关键证据）

**本院查明**：
- 关键事实（按时间顺序梳理关键节点）
- 关键时间节点（合同签订、纠纷发生、起诉等）
- 关键证据（法院据以认定事实的证据）
- 争议事实（双方有争议的事实点）

**证据认定**：
- 关键证据（按证据编号或提交方分类）
- 法院认定意见（采信/不采信及理由）
- 证明力评估（强/中/弱）

### 裁判理由抽取

**争议焦点识别**：
- 焦点编号（按顺序编号：焦点一、焦点二...）
- 焦点标题（法院归纳的争议焦点）
- 原告观点（原告在该焦点上的主张和理由）
- 被告观点（被告在该焦点上的抗辩和理由）
- 法院认定（法院对该焦点的认定结果）⭐
- 事实依据（法院认定的事实基础）
- 法律依据（引用的法条名称和条款）
- 说理逻辑（法院的推理过程）
- 裁判结论（该焦点的最终认定结果）

**法律依据提取**：
- 法律名称（引用的法律法规名称）
- 条款号（具体条款号）
- 条文内容（关键条文内容）
- 司法解释（引用的司法解释）

### 裁判结果抽取

**判决主文**：
- 判决项编号（按顺序编号 1、2、3...）
- 判决内容（每项判决的具体内容）
- 履行义务人（承担义务的一方）
- 履行期限（判决生效后的履行期限）
- 履行方式（金钱给付/行为履行/确认等）

**诉讼费用**：
- 案件受理费（具体金额和负担方）
- 其他费用（保全费、鉴定费等）
- 负担方式（费用的承担方式）

**上诉权利**：
- 上诉期限（通常为15日）
- 上诉法院（上级法院名称）

---

## 处理流程

### 一审判决处理流程

1. **基础信息提取**：从前 10-20 行提取案号、法院、当事人等信息
2. **事实认定提取**：提取原告诉称、被告辩称、本院查明、证据认定
3. **裁判理由提取**：提取争议焦点、法律依据、说理逻辑
4. **裁判结果提取**：提取判决主文、诉讼费用、上诉权利
5. **质量检查**：完整性、准确性、逻辑性检查

## 输出侧重

### 内部版

- 📋 核心要点速览
- 完整的事实认定链条
- 争议焦点与法院认定（逐一对应）
- 判决结果详解
- 法律分析与策略建议
- 证据详细分析（可选）
- 裁判要点与实务启示

### 研究版

- 识别的法律问题
- 上诉策略研究课题
- 改判可能性研究
- 类似案例检索方向

### 客户版

- 判决结果通俗解释
- 上诉利弊分析
- 执行建议
- 风险提示
- 建议行动

## 抽取质量检查

### 完整性检查

- [ ] 基础信息：案号、法院、当事人、日期齐全
- [ ] 事实认定：原告诉称、被告辩称、本院查明完整
- [ ] 裁判理由：全部争议焦点已提取
- [ ] 裁判结果：判决主文逐项列出，无遗漏

### 准确性检查

- [ ] 案号格式正确（"(20XX)X民初XXXX号"）
- [ ] 法院名称完整（包含省市区）
- [ ] 当事人信息准确（原告/被告不混淆）
- [ ] 法条引用完整（法律名称+条款号）
- [ ] 判决项与原文一致（无遗漏或错位）

### 逻辑性检查

- [ ] 事实认定与争议焦点对应
- [ ] 法律依据与争议焦点匹配
- [ ] 判决结果与裁判理由一致
- [ ] 时间线逻辑连贯（无矛盾的时间点）
