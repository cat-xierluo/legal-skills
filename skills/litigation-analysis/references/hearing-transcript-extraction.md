# 庭审笔录处理文档

本文档定义了庭审笔录的完整处理流程，包括识别特征、抽取规范、处理流程和输出侧重。

---

## 识别特征

### 来源形式

- **录音转写**：来自庭审录音的自动转写（如 FunASR）
- **人工记录**：庭审现场的人工记录
- **法院系统导出**：从法院电子系统导出的官方记录

### 内容特征

- **对话形式**：法官、原告/申请人、被告/被申请人、代理人等多角色对话
- **时间顺序**：按庭审流程推进（开庭准备 → 法庭调查 → 法庭辩论 → 最后陈述）
- **口语化**：包含口语化表达、重复、语气词
- **非结构化**：通常没有固定的章节结构

---

## 抽取规范

庭审笔录用于庭审后的复盘分析和策略制定，采用结构化深度抽取。

### 抽取原则

- **信息密集**：包含大量庭审过程中的关键信息和细节
- **多角色交互**：需要识别不同角色的发言
- **策略导向**：最终目的是为后续策略调整提供依据

抽取时需要：角色识别、信息归类、策略提炼

### 基础信息抽取

| 字段 | 提取规则 | 说明 |
|------|---------|------|
| 案号 | 从笔录开头或法官陈述中提取 | 如"(20XX)X民初XXXX号" |
| 开庭时间 | 从笔录开头提取 | YYYY-MM-DD 格式 |
| 审理法院/仲裁庭 | 从笔录中提取完整名称 | 包含省市区信息 |
| 主审法官/仲裁员 | 从笔录中提取姓名 | 用于后续沟通 |
| 我方当事人 | 识别原告/被告/申请人/被申请人 | 区分我方地位 |
| 对方当事人 | 识别对方当事人及其代理人 | 用于分析对方策略 |
| 争议焦点 | 从法官归纳或双方辩论中提取 | 庭审核心议题 |

### 庭审流程抽取

**开庭准备**：
- 身份核对（当事人、代理人身份信息）
- 权利告知（回避权、辩论权等）
- 合议庭组成（法官、陪审员、书记员）

**法庭调查**：
- 原告主张（诉讼请求、事实依据）
- 被告答辩（承认/否认事实、抗辩理由）
- 证据展示（双方证据的编号、名称、证明目的）
- 质证情况（对方对证据的质证意见）

**法庭辩论**：
- 争议焦点（法庭归纳的争议焦点）
- 辩论观点（双方对各焦点的辩论）
- 法官提问（法官的关键提问及双方回应）

**最后陈述**：
- 原告最后陈述（核心观点总结）
- 被告最后陈述（核心观点总结）
- 法庭调解（是否接受调解的意愿）

### 关键信息抽取

**证据与质证**：
- 我方证据（编号、名称、证明目的、对方质证意见）
- 对方证据（编号、名称、证明目的、我方质证意见）
- 争议证据（双方有争议的证据）
- 法院态度（法官对证据的关注点和倾向）

**法官提问**：
- 关键问题（法官提出的问题）
- 我方回应（我方的回答要点）
- 对方回应（对方的回答要点）
- 法官倾向（问题可能透露的法官倾向）

**程序事项**：
- 法官要求（补充证据、提交意见等）
- 时限安排（各项要求的截止时间）
- 下次开庭（如有再次开庭的安排）

## 处理流程

### 庭审笔录处理流程

1. **基础信息提取**：从前 10-20 行提取案号、时间、法院、当事人等
2. **角色识别**：识别发言角色（法官、原告、被告、代理人等）
3. **流程梳理**：按庭审流程（调查→辩论→陈述）梳理内容
4. **焦点归纳**：提取法庭归纳的争议焦点
5. **证据整理**：整理双方证据及质证情况
6. **提问提炼**：提炼法官关键提问及各方回应
7. **策略分析**：基于庭审情况分析策略调整方向
8. **质量检查**：完整性、准确性、逻辑性检查

## 抽取质量检查

### 完整性检查

- [ ] 基础信息：案号、开庭时间、法院、当事人齐全
- [ ] 争议焦点：法庭归纳的争议焦点已提取
- [ ] 证据情况：双方证据及质证情况完整
- [ ] 法官提问：关键提问及各方回应已记录
- [ ] 时限要求：法官要求与时限安排已整理

### 准确性检查

- [ ] 案号格式正确
- [ ] 开庭时间准确（具体到日）
- [ ] 当事人地位准确（原告/被告不混淆）
- [ ] 证据编号与名称一致
- [ ] 质证意见与发言主体对应

### 逻辑性检查

- [ ] 争议焦点与双方主张对应
- [ ] 证据与证明目的匹配
- [ ] 法官提问与争议焦点关联
- [ ] 时限安排时间逻辑合理

## 使用说明

### 与其他文档的配合

庭审笔录分析可与以下文档配合使用：

1. **起诉状/答辩状**：了解庭前双方主张
2. **证据清单**：对比庭前证据与庭审举证
3. **代理词**：了解庭审后的书面陈述
4. **判决书**：对比庭审情况与最终判决

### 决策支持

庭审笔录分析主要用于：

- **策略调整**：根据庭审情况调整诉讼策略
- **证据补强**：识别需要补充的证据
- **法律研究**：针对庭审中新出现的法律问题进行研究
- **客户沟通**：向客户汇报庭审情况和下一步计划
- **团队复盘**：内部团队复盘庭审表现和案件走向

## 核心原则

### 信息处理原则

- **只做提取**：只做信息提取、归纳、结构化与行动项整理
- **不新增事实**：不新增事实，不虚构法条或裁判观点
- **缺失标注**：用「[待确认]」标注缺失信息，不得编造
- **客观记录**：客观记录庭审过程，避免主观评价

### 保密与脱敏

- **内部使用**：案件信息敏感，建议保留完整信息供内部使用
- **对外脱敏**：对外版本需进行脱敏处理
- **谨慎分享**：不随意向无关人员泄露庭审细节
