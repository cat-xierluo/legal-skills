# 领域扩展指南

本文档说明如何在通用处理框架基础上，为特定法律领域添加专用分析维度。

## 设计理念

**通用框架 + 专用领域**

- **通用框架**：适用于所有案件的基础处理能力（文书识别、信息提取、三层输出）
- **专用领域**：针对特定法律领域的深度分析维度（专业术语、特殊要点、领域法条）

**优势**:
- 保持核心框架的通用性和稳定性
- 各领域独立扩展，互不影响
- 新增领域无需修改核心代码

## 现有领域示例

### 知识产权与不正当竞争

**文件**: `references/ip-case-dimensions.md`

**包含内容**:
- 6种案件类型（专利权、商标权、著作权、商业秘密、不正当竞争、特许经营）
- 每种类型的专用分析维度
- 相关法条提示
- 常见争议焦点

**使用方式**:
当案件被识别为IP或不正当竞争类型时，在通用分析基础上，额外应用该领域的专用分析维度。

## 如何添加新领域

### 步骤1：创建领域维度文件

在 `references/` 目录下创建新文件：

```
references/[领域名称]-case-dimensions.md
```

**命名规范**:
- `labor-case-dimensions.md` - 劳动法
- `company-case-dimensions.md` - 公司法
- `contract-case-dimensions.md` - 合同法
- `tort-case-dimensions.md` - 侵权法

### 步骤3：实际案例 - 劳动法领域

**文件**: `references/labor-case-dimensions.md`

```markdown
# 劳动法案件专用分析维度

## 案件类型识别

**关键词库**:
- 劳动关系确认：劳动关系、劳动合同、用工关系、事实劳动关系
- 违法解除：解除劳动合同、违法解除、经济补偿金、赔偿金
- 工伤认定：工伤、工伤保险、伤残等级、停工留薪
- 加班费：加班、加班费、延时工资、值班

## 案件类型1：劳动关系确认纠纷

### 专用分析维度

#### 1. 劳动关系构成要件分析

**核心问题**: 双方是否构成劳动关系

**分析要点**:
- 主体资格：用人单位是否具备用工主体资格，劳动者是否达到法定年龄
- 从属性特征：是否接受管理、是否遵守规章制度、是否提供有偿劳动
- 人格从属：是否纳入用人单位组织体系
- 经济从属：报酬支付方式、是否承担经营风险

**法律依据**:
- 《劳动合同法》第7条
- 《关于确立劳动关系有关事项的通知》（劳社部发[2005]12号）

**常见争议焦点**:
- 是否存在"管理与被管理"关系
- 工资支付记录的证明力
- 工作证、考勤记录等证据的效力

#### 2. 证据链完整性评估

**核心问题**: 劳动关系证据是否充分

**关键证据类型**:
- 工资支付凭证（银行流水、工资条）
- 考勤记录
- 工作证、工牌、工作服
- 社保缴纳记录
- 同事证言
- 工作邮件、微信/钉钉聊天记录

**证据强度排序**:
1. 书面劳动合同（最强）
2. 工资支付凭证 + 社保记录
3. 考勤记录 + 工作证
4. 证人证言（最弱）

### 研究课题方向

**高频研究课题**:
- 新业态用工（网约车、外卖）的劳动关系认定
- 退休返聘人员的关系性质
- 实习生劳动关系认定
- 非全日制用工的特殊规定

**检索关键词建议**:
- "劳动关系 认定标准"
- "新业态 用工关系"
- "事实劳动关系 证据"

---

## 案件类型2：违法解除劳动合同纠纷

### 专用分析维度

#### 1. 解除合法性分析

**核心问题**: 用人单位解除是否符合法定情形

**法定解除情形**:
- 过错性解除（劳动合同法第39条）
- 非过错性解除（劳动合同法第40条）
- 经济性裁员（劳动合同法第41条）

**分析框架**:
```
解除程序是否合法
  ├── 是否履行通知义务
  ├── 是否履行协商义务
  └── 是否支付经济补偿

解除实体是否合法
  ├── 是否符合法定解除情形
  ├── 证据是否充分
  └── 是否存在禁止解除情形（孕期、医疗期等）
```

#### 2. 经济补偿金/赔偿金计算

**计算规则**:
- 经济补偿金 = 工作年限 × 月工资
- 违法解除赔偿金 = 2 × 经济补偿金

**争议要点**:
- 月工资基数的确定（是否包含奖金、津贴）
- 工作年限的计算（跨单位连续工龄）
- "双倍封顶"规则的适用

**法律依据**:
- 《劳动合同法》第47条、第87条
- 《劳动合同法实施条例》第27条

### 研究课题方向

**高频研究课题**:
- "严重违反规章制度"的认定标准
- 医疗期满解除的合法性
- 末位淘汰制的效力
- N+1补偿的适用情形
```

## 扩展注意事项

### 1. 保持独立性

- 每个领域维度文件应独立完整
- 不依赖其他领域文件
- 可以单独使用或组合使用

### 2. 结构一致性

- 遵循统一的文档结构模板
- 使用相同的章节标题风格
- 保持分析维度的层次清晰

### 3. 实用性优先

- 关注律师实际工作中的常见问题
- 提供可操作的分析框架
- 包含具体的法条和案例引用

### 4. 定期更新

- 法律法规变更时及时更新
- 补充新的司法解释和指导案例
- 基于实际使用反馈优化维度

## 领域识别机制

### 自动识别

系统通过关键词库自动识别案件所属领域：

```
案件文本 → 关键词匹配 → 识别领域 → 应用专用维度
```

### 多领域叠加

一个案件可能同时涉及多个领域：

**示例**：公司高管劳动争议案件
- 应用：劳动法维度 + 公司法维度
- 分析：劳动关系认定 + 竞业限制 + 股权激励

### 领域优先级

当多个领域匹配时，按案由主要性确定优先级：

1. 案由明确指向的领域（最高优先级）
2. 关键词频率最高的领域
3. 争议焦点涉及的领域

## 质量标准

新增领域维度文件应满足：

- **完整性**：覆盖该领域主要案件类型（至少3-5类）
- **专业性**：分析维度准确反映法律实务要点
- **可操作性**：提供明确的分析框架和判断标准
- **法律依据**：引用准确的法条和司法解释
- **研究导向**：提供有价值的研究课题方向
