# legal-case-analysis 试跑评估

**评估时间：** 2026-06-05  
**试跑材料：** 竞赛商品房买卖合同纠纷材料包  
**输出样例：** `examples/contest-sample-analysis.md`

## 一、总体评价

当前 `legal-case-analysis` 已经能支撑一次完整的通用法律分析试跑。它的主流程覆盖了案件画像、事实证据台账、问题树、检索结果吸收、逐项论证、风险评估和策略建议。用本次复杂民商事赛题测试后，报告结构基本可用，且能体现“通用 Skill + 具体案由动态适配”的设计目标。

但试跑也暴露出一个核心问题：当前 Skill 对“分析过程”的要求较清楚，对“中间产物如何保存、检索结果如何回填、金额如何计算、时间效力如何切换”的规范还不够细。正式竞赛或教学使用时，建议先迭代到 0.2.0，再让学员参考。

## 二、表现较好的部分

### 1. 通用定位成立

本案虽然是商品房买卖合同纠纷，但实际报告没有把 Skill 写死成商品房专用。报告中的底层结构仍然是：

- 委托目标。
- 事实和证据。
- 法律问题树。
- 检索结果。
- 逐项论证。
- 风险等级。
- 行动建议。

这说明 Skill 的通用框架可迁移到其他案件。

### 2. 事实和证据区分有效

试跑中可以自然区分：

- 合同、付款、贷款、查封等客观材料。
- 买受人陈述。
- 开发商回函和争议性文件。
- 待补充的官方资料。

这一点对法律分析类 Skill 很关键，也契合竞赛评分中的“事实梳理”和“证据分类”要求。

### 3. 检索闭环方向正确

报告能够把本地检索存档整理成规则表，而不是直接堆法条。尤其能识别：

- 解除权应区分合同约定解除和司法解释催告规则。
- 贷款处理不能简单把全部房款返还给买受人。
- 抵押争点不能误用旧版一倍赔偿规则。
- 执行救济和合同解除诉讼是并行路径。

### 4. 风险评估有实务价值

试跑报告不仅给结论，还能给出风险来源和补强动作，例如银行参诉、抵押知情确认书原件、家庭住房情况、执行处置进展等。

## 三、暴露的问题

### 1. 缺少“已有检索结果回填”规则

当前 Skill 只写了“生成法律检索任务”和“吸收检索结果”，但没有说明当用户已经提供检索存档时，应如何处理。

本次试跑实际做法是把 `法律检索存档/` 当作检索结果回填，但这是临时判断。后续应增加：

- 检索状态标记：未检索、已检索待校验、已校准。
- 检索结果来源等级：官方法条、案例摘要、研究报告、命题校准文件。
- 过时规则拦截：发现旧法或已删除规则时自动标注禁用。

### 2. 缺少“时间效力切换”模块

本案同时有 2024 年首次委托时点、2026 年竞赛作答时点、2025 年执行异议之诉司法解释施行时点。当前 Skill 只提醒“注意新旧法”，但没有提供固定处理表。

建议新增 `references/time-effectiveness-check.md`，要求输出：

| 时间基准 | 可用规则 | 不可直接适用规则 | 报告表达 |
|----------|----------|------------------|----------|

### 3. 金额计算模板不足

本案涉及首付款、已还本金、已还利息、尚欠贷款、资金占用损失、逾期交房违约金、诉讼费用、鉴定费用。当前模板只要求“请求或方案评估”，没有专门的金额台账。

建议新增 `templates/damages-calculation-table.md`，区分：

- 返还本金。
- 银行贷款清偿。
- 已偿还贷款本息。
- 违约金。
- 资金占用损失。
- 费用类损失。
- 不得重复计算的项目。

### 4. 程序路径模板还不够强

本案同时存在合同纠纷诉讼和执行程序救济。当前模板能写出来，但没有强制区分“主路径、并行路径、后续路径”。

建议新增 `references/procedure-path-analysis.md`，要求输出：

- 路径目的。
- 启动条件。
- 诉讼主体。
- 请求类型。
- 证据条件。
- 与其他路径的关系。

### 5. 缺少“脱敏输出模式”

用户要求去除个人信息后，本次试跑报告主动使用了角色称谓，但 Skill 本身没有这个功能说明。

建议在 `SKILL.md` 增加可选规则：

- 如用于课程、竞赛、公开演示，默认匿名化当事人、公司名、联系方式、地址、身份识别字段、账号等。
- 保留法律关系和事实结构。
- 使用“委托人A、相对方B、第三人C、机构D”等称谓。

### 6. 中间产物不够分层

当前输出是一份完整报告。实际教学和竞赛展示更需要展示过程：

- 材料清单。
- 事实台账。
- 法律问题树。
- 检索任务清单。
- 检索结果回填表。
- 最终报告。

建议新增 `templates/run-log.md` 或 `templates/analysis-workbook.md`，用于展示 AI 执行过程。

## 五、结论

当前版本适合作为“通用法律分析 Skill 初稿”和课程演示基础，但如果要在竞赛中体现 Skill 设计优势，建议优先补强“检索结果回填、时间效力、金额计算、程序路径、脱敏输出、中间产物展示”六个环节。

本次试跑证明：Skill 的方向是对的，但还需要把律师实务中最容易出错的几个环节做成强约束模板。
