# 报告详略控制规则

本文件用于在生成法律分析前判断案件复杂度，并据此选择输出深度。目标是避免简单咨询被写成冗长报告，也避免复杂案件未经案情分析和法律检索就直接输出结论。

## 一、先判断输出场景

收到案件材料后，先判断用户真正需要的是哪一类输出：

| 输出场景 | 典型情况 | 输出重点 |
|----------|----------|----------|
| 简单咨询 | 自然人日常咨询、事实较少、目标单一、金额或风险较低 | 结论、理由、风险、下一步 |
| 标准案件分析 | 有合同、证据、相对方争议或初步诉讼/谈判需求 | 事实证据、争点、检索、法律分析、策略 |
| 复杂案件分析 | 主体多、事实链长、金额大、程序交叉、证据争议大或需要正式策略 | 分阶段分析、检索策略、规则回填、内部报告/客户报告 |

如果无法判断复杂度，先输出“案情画像和材料缺口”，不要直接生成最终法律分析报告。

## 二、简单咨询

适用于：

- 事实关系清楚。
- 争议目标单一。
- 材料较少。
- 用户只需要初步判断或行动建议。
- 不需要完整诉讼策略。

默认使用 `templates/simple-consultation-answer.md` 输出“简版法律分析/咨询答复”。该模板只保留咨询问题、已知事实、初步结论、主要理由、风险提示、下一步建议和需补充材料。

### 简单咨询是否需要检索

以下情形可以先不做完整检索，但仍应避免虚构法条：

- 用户只需要常识性、方向性判断。
- 不引用具体法条条号。
- 不涉及诉讼时效、程序期限、权利顺位、强制性规则或新旧法问题。

以下情形即使是简单咨询，也应先做法律检索：

- 需要引用具体法条、司法解释或类案。
- 涉及时效、除斥期间、上诉/复议/异议期限。
- 涉及管辖、举证责任、保全、执行、优先权。
- 用户将把答复用于对外沟通、谈判、投诉或诉讼。
- AI 对规则准确性没有把握。

## 三、标准案件分析

适用于：

- 已有基本案件材料和证据。
- 存在明确相对方和争议焦点。
- 用户需要起诉、应诉、谈判、风险评估或客户汇报。
- 案件不至于复杂到需要多阶段拆解，但不能只用咨询答复处理。

默认流程：

1. 了解基础案情：主体、目标、程序阶段、关键事实、证据和争议。
2. 拆解法律问题：法律关系、请求权基础、抗辩风险、程序路径。
3. 制定检索策略：法条、司法解释、类案、时效、举证责任、程序规则。
4. 回填检索结果：规则摘要、适用条件、本案对应事实、风险提示。
5. 形成法律分析：事实分析、证据分析、法律适用、风险评估。
6. 输出报告：使用 `templates/legal-analysis-report.md` 或 `templates/client-summary.md`。

标准案件分析可以一次性输出完整报告，但报告中必须保留：

- 待核实事实。
- 证据缺口。
- 检索结果来源。
- 对方可能抗辩。
- 风险等级和补强建议。

## 四、复杂案件分析

适用于：

- 主体多或法律关系交叉。
- 事实链长，时间跨度大。
- 证据种类多，真实性或证明力争议明显。
- 涉及多个程序路径，如诉讼、仲裁、保全、执行、行政、破产等。
- 涉及高金额、高风险或需要客户正式决策。
- 涉及新旧法、裁判规则分歧或强烈依赖类案。

复杂案件不要直接生成最终报告。应分阶段输出：

1. 案情画像和材料缺口。
2. 事实与证据台账。
3. 法律关系、请求权基础和争议焦点。
4. 法律检索策略。
5. 检索结果回填和规则适用。
6. 内部法律分析报告。
7. 客户版法律分析报告或客户沟通摘要。

## 五、内部版与客户版

### 内部版

面向律师团队或办案人员，可以保留较完整的推演过程：

- 事实和证据台账。
- 请求权基础和构成要件。
- 正反两方面法律规则。
- 类案差异。
- 对方抗辩。
- 证据缺口。
- 程序选择。
- 风险假设。
- 诉讼或谈判策略。

内部版应更完整，不怕复杂，但要结构清楚。

### 客户版

面向非法律专业客户，应更注重结论和行动：

- 结论靠前。
- 少写抽象法理。
- 用通俗语言解释风险。
- 明确“现在该做什么”。
- 对不确定事项说明原因。

客户版不应省略关键风险，但可以压缩推理论证过程。

## 六、输出选择规则

| 判断结果 | 推荐输出 | 推荐模板 |
|----------|----------|----------|
| 简单咨询 | 简版法律分析/咨询答复 | `templates/simple-consultation-answer.md` |
| 标准案件 | 法律分析报告或客户说明 | `templates/legal-analysis-report.md` / `templates/client-summary.md` |
| 复杂案件第一阶段 | 案情画像、材料缺口、检索策略 | `templates/legal-research-query-list.md` + 自定义阶段性说明 |
| 复杂案件最终阶段 | 内部法律分析报告或客户版报告 | `templates/legal-analysis-report.md` / `templates/client-summary.md` |

## 七、降级与升级

分析过程中可以调整复杂度：

- 简单咨询中发现事实争议、证据缺口或程序风险，应升级为标准案件分析。
- 标准案件中发现多法律关系、多程序路径或重大争议，应升级为复杂案件分阶段分析。
- 复杂案件中某个单点问题可单独降级为简版专项答复，但不能替代总报告。

## 八、质量要求

无论输出长短，都必须做到：

- 不把单方陈述写成确定事实。
- 不把未检索规则写成确定法律依据。
- 不用“必胜”“一定支持”等承诺式表达。
- 对待补充材料和风险保留明确提示。
- 结论与事实、证据、规则之间能相互对应。
