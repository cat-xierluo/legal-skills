# 复杂民商事案件法律分析步骤

本文件用于补足“法律检索之外”的实体分析流程。`legal-case-analysis` 不应只做材料摘要，也不应只把检索结果堆进报告。复杂民商事案件应按以下步骤推进。

## 一、确认分析任务

先回答三个问题：

| 问题 | 说明 |
|------|------|
| 谁委托 | 原告、被告、第三人、案外人、交易方、管理方等 |
| 要什么结果 | 起诉、应诉、解除、履行、返还、赔偿、排除执行、谈判、合规整改等 |
| 到什么阶段 | 咨询、诉前、一审、二审、执行、再审、非诉交易、争议爆发前 |

不同阶段的分析重点不同。诉前重请求和证据，诉中重争点和抗辩，执行阶段重权利顺位和程序条件。

## 二、定位法律关系

先判断案件涉及哪些法律关系，不要直接套案由。

常见类型：

- 合同关系：买卖、借款、租赁、服务、承揽、建设工程、担保。
- 物权关系：所有权、抵押权、预告登记、查封、共有。
- 侵权关系：人身、财产、网络、知识产权、商业侵权。
- 公司关系：股权、控制权、董事高管责任、股东知情权。
- 程序关系：保全、执行异议、执行异议之诉、破产、仲裁。

输出时说明：

| 法律关系 | 主体 | 依据材料 | 对分析的影响 |
|----------|------|----------|--------------|

## 三、确定主张或防御目标

从委托人角度列出可能目标：

| 目标类型 | 示例 |
|----------|------|
| 确认类 | 确认合同解除、确认权利、确认债权 |
| 给付类 | 返还款项、支付违约金、赔偿损失 |
| 行为类 | 交付标的、办理登记、配合注销 |
| 形成类 | 解除合同、撤销行为、变更关系 |
| 程序类 | 财产保全、执行异议、排除执行、参与分配 |
| 谈判类 | 和解、重组、延期、分期、替代履行 |

每个目标都应转化为具体请求或策略动作。

## 四、拆解请求权基础

对每项主张按“请求权基础 -> 构成要件 -> 事实 -> 证据 -> 风险”处理。

| 请求/抗辩 | 请求权基础 | 构成要件 | 已有事实 | 支持证据 | 缺口 |
|-----------|------------|----------|----------|----------|------|

如果无法找到请求权基础，应标注“法律依据待检索”，并交给 `yuandian-law-search` 检索。

## 五、建立事实与要件映射

不要只写时间轴。要把事实放到构成要件中验证：

| 构成要件 | 支持事实 | 支持证据 | 反向事实/抗辩 | 初步判断 |
|----------|----------|----------|----------------|----------|

这样可以避免报告出现“事实很多，但结论和事实脱节”的问题。

## 六、识别争议焦点

争议焦点不是材料摘要，而是会影响结果的判断点。

常见争点类型：

- 法律关系是否成立。
- 义务是否到期。
- 是否构成违约或侵权。
- 是否存在免责或抗辩事由。
- 损失范围和计算方式。
- 证据真实性和证明力。
- 程序路径和主体资格。
- 权利顺位和执行保护。

输出格式：

| 争议焦点 | 影响结果 | 我方观点 | 对方可能观点 | 需检索问题 | 需补强证据 |
|----------|----------|----------|--------------|------------|------------|

## 七、先检索，再论证

争议焦点确定后，先用 `yuandian-law-search` 检索法律依据和类案，再进入论证。

检索顺序建议：

1. 现行有效法条和司法解释。
2. 关键条文的适用条件。
3. 同类案件裁判规则。
4. 程序性规则。
5. 新旧法和时间效力。
6. 反向案例或不支持规则。

检索完成后，必须把检索结果回填到争点表，说明规则如何适用于本案事实。

## 八、形成抗辩与反驳

对每个主张至少预判三类抗辩：

| 抗辩类型 | 示例 |
|----------|------|
| 事实抗辩 | 事实不存在、时间不对、因果关系不足 |
| 法律抗辩 | 不符合构成要件、法律依据错误、规则已失效 |
| 证据抗辩 | 无原件、来源不明、证明目的不成立 |
| 程序抗辩 | 管辖、时效、主体、期间、前置程序 |

每项抗辩都要对应反驳路径和补证动作。

## 九、评估风险和方案

不要只给“能不能赢”的结论。应拆成：

- 法律依据风险。
- 事实认定风险。
- 证据证明风险。
- 程序推进风险。
- 执行回收风险。
- 谈判成本风险。

输出格式：

| 方案 | 收益 | 风险 | 前提条件 | 推荐程度 |
|------|------|------|----------|----------|

## 十、形成交付报告

最终报告应体现分析链条：

```text
委托目标
  -> 法律关系
  -> 请求/抗辩
  -> 构成要件
  -> 事实证据
  -> 法律检索
  -> 适用分析
  -> 风险评估
  -> 策略建议
```

如果报告中某一结论没有对应的事实、证据或检索依据，应退回前序步骤补充。
