# 迭代上下文

本文件记录 `legal-case-analysis` 的长期设计上下文，供后续持续迭代时快速理解边界、资源分层和优先方向。

## 一、定位

`legal-case-analysis` 是通用法律分析 Skill，用于把案件材料、咨询材料、证据材料和法律检索结果转化为可复用的法律分析结果、风险评估、诉讼策略、客户说明或结构化法律分析报告。

本 Skill 的核心定位是法律分析引擎。报告只是其中一种交付形态；在服务方案、客户沟通、诉讼策略、合同风险、结案复盘等场景中，法律分析过程都可以作为前置步骤存在。

本 Skill 不绑定具体案由。具体领域规则通过案件材料、法律检索结果和用户目标动态补足。

## 二、核心问题

本 Skill 解决的是律师办案中的通用分析流程问题：

1. 材料如何转化为事实和证据台账。
2. 事实如何映射到请求权基础和构成要件。
3. 争议焦点如何转化为法律检索任务。
4. 检索结果如何回填为可适用的法律规则。
5. 法律规则如何结合事实、证据和抗辩形成结论。
6. 风险、补证和程序路径如何转化为可执行建议。
7. 分析结论如何作为其他 Skill 的输入，而不是被误认为必须生成最终报告。

## 四、非目标

本 Skill 不做以下事情：

- 不替代 OCR 或文档转换工具。
- 不替代法律检索工具。
- 不直接生成 Word 排版文件。
- 不替代 `legal-proposal-generator` 生成诉讼服务方案、非诉服务方案、法律服务建议书等服务文档。
- 不基于无材料支撑的抽象法律问题输出确定结论。
- 不把单一案由的专项规则写入核心适用边界。

## 五、法律检索原则

涉及具体法条、司法解释、类案、时间效力、时效、管辖、举证责任、执行异议、优先权、损失计算等问题时，应先生成法律检索任务，再基于检索结果分析。

法律检索结果应整理为规则表，说明：

- 来源。
- 规则摘要。
- 适用条件。
- 本案对应事实。
- 支持方向。
- 风险提示。

禁止只堆法条或只凭模型记忆写确定法律结论。

## 六、输出风格

法律分析报告应做到：

- 事实、证据、规则和结论可追溯。
- 有利因素和不利因素并列呈现。
- 风险使用“高/中/低 + 理由”表达。
- 对未提及或待核实事项明确标注。
- 对程序路径和补证动作给出可执行建议。

客户说明应做到：

- 表达通俗。
- 结论靠前。
- 风险明确。
- 行动建议具体。

## 刑事犯罪论体系决策：不法—有责二阶层（2026-07-20）

本节记录刑事支架犯罪论体系的体系选择与学理来源，供后续正文改造和迭代时追溯。完整决策另见 `DECISIONS.md` 2026-07-20 条目。

### 体系选择

采用**不法—有责二阶层**（张明楷立场，亦为 2025 年法考刑法采用的标准体系），先客观不法后主观有责，层层递进、任一层不过即出罪。完全移除四要件作为主干。

- **不法阶层**（以客观为原则，含主观不法要素）：客观构成要件（行为、结果、因果关系、对象、身份）+ 主观不法要素（目的犯的目的、倾向、表现犯心理；无此要素的罪名跳过）+ 违法阻却（正当防卫、紧急避险、承诺、法令行为、自救、义务冲突）。
- **有责阶层**：责任能力（年龄、精神状态）+ 责任形式（故意/过失）+ 责任阻却（违法性认识可能性、期待可能性；通常默认具备、有争议才展开）。
- **共犯**：正犯/共犯二分 + 限制从属（共犯以正犯有客观不法为前提，不要求正犯有责）。

### 关键学理点（避免后续改错）

1. **目的犯的目的是主观不法要素，属不法层**（不是责任要素）；故意/过失才是责任要素（属有责层）。两者不可错置。
2. **不法非纯客观**：以客观为原则，但目的、倾向、表现犯心理等主观要素在不法层评价。
3. **故意的地位学界有争论**：张明楷（本 skill 跟随）= 责任要素；周光权/劳东燕 = 构成要件要素；江溯 = 拆分心理性故意（不法）/罪责故意+不法意识（有责）。本 skill 跟随二阶层通说，故意/过失放有责层。
4. **二阶层 vs 三阶层无本质区别**：仅"构成要件该当性"是否并入"违法性"。二阶层是该当性并入违法性合成"不法"，更简洁。
