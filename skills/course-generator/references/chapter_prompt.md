# 专题文档撰写提示词

基于课程大纲和分析素材，生成专题详细内容。

## 撰写要求

1. **忠实原文**：保留文献中的核心观点、具体例子、重要表述
2. **叙述性写作**：完全避免要点罗列，使用段落形式，包含完整的论证和阐述
3. **详尽完整**：每个概念都有完整的解释和论证，保持逻辑连贯
4. **避免外部扩展**：不添加文档外的知识和信息
5. **自然表达**：
   - 禁止使用"原文中"、"根据原文"、"主讲人强调"、"主讲人指出"、"主讲人提到"、"主讲人认为"、"发言人表示"等表述
   - 直接陈述内容本身，不强调内容来源归属
   - 将转录内容转换为客观的知识陈述形式
   - 去除发言人标记，将内容整合为连贯的叙述

## 叙述性写作原则

**重要：完全避免要点列表**

- 使用段落形式展开所有内容
- 每个概念都有完整的解释和论证
- 保持逻辑连贯，层层递进
- 避免使用"首先、其次、最后"等机械连接词
- 用自然流畅的叙述方式组织内容

## 章节模块库

根据内容特点，从以下模块中选择合适的组合：

### 可选模块

| 模块名称 | 适用场景 | 编号格式 |
|---------|---------|---------|
| 引言与概述 | 需要背景介绍和内容导航 | 一、 |
| 核心概念与原理 | 理论性内容，需要概念阐释 | 二、 |
| 理论框架 | 学术性、理论性较强的内容 | 三、 |
| 历史背景 | 需要时间线或发展历程的内容 | 四、 |
| 实践案例 | 有具体应用实例的内容 | 五、 |
| 应用场景 | 需要场景说明的内容 | 六、 |
| 最佳实践 | 需要操作指南的内容 | 七、 |
| 对比分析 | 需要比较不同方案/观点的内容 | 八、 |
| 未来展望 | 有前瞻性、趋势性内容 | 九、 |
| 本章小结 | 需要总结回顾的内容 | 十、 |
| 延伸阅读 | 有相关资源推荐的内容 | 十一、 |

### 模块选择原则

1. 根据内容类型选择合适的模块
2. 每章至少包含 2 个模块
3. 模块顺序遵循逻辑关系
4. 避免内容空洞的模块

### 默认推荐组合

- **理论类**：引言与概述 + 核心概念与原理 + 本章小结
- **实践类**：引言与概述 + 应用场景 + 最佳实践 + 本章小结
- **案例类**：引言与概述 + 实践案例 + 对比分析 + 本章小结
- **综合类**：根据内容特点灵活组合

### 模块详细说明

#### 模块1：引言与概述

```
## 一、引言与概述

本章将深入探讨[章节主题]的核心概念和实践应用。首先介绍该主题的背景和重要性，帮助读者建立整体认知框架。接着概述本章将要讨论的主要内容，为读者提供清晰的学习路径。
```

#### 模块2：核心概念与原理

```
## 二、核心概念与原理

[详细阐述核心概念，采用叙述性表达，每个概念都有完整的解释和论证。以段落形式展开说明，避免使用要点列表。整个论述过程要保持逻辑连贯，层层递进。]

### 1、[子主题1]

[以段落形式深入讲解，系统阐述该子概念的各个方面。首先介绍其基本定义和内涵，然后探讨其起源和发展历程，分析其理论基础和核心原理，最后说明其实际意义和应用价值。]

### 2、[子主题2]

[继续以叙述方式展开，与前一子主题形成逻辑递进关系。深入剖析该概念的本质特征，探讨其与相关概念的内在联系。]
```

#### 模块3：理论框架

```
## 三、理论框架

[阐述本主题的理论基础和框架结构，以段落形式说明核心理论的构成要素、内在逻辑和理论来源。]
```

#### 模块4：历史背景

```
## 四、历史背景

[以时间线或发展历程的方式，介绍该主题的历史沿革、发展阶段和重要节点。以叙述方式展开，避免使用要点列表。]
```

#### 模块5：实践案例

```
## 五、实践案例

通过具体案例来展示理论知识的实际应用，帮助读者将抽象概念转化为具体认知。

### 案例一：[案例名称]

这个案例生动地展示了[章节主题]在实际工作中的具体应用。案例背景是[背景描述]，面临的主要挑战是[挑战描述]。通过采用[具体方法]，成功实现了[预期效果]。整个过程中遇到的关键问题包括[问题清单]，而解决方案则体现了[章节主题]的核心价值。

### 案例二：[案例名称]

作为另一个典型应用场景，这个案例从不同角度展示了[章节主题]的灵活性和适用性。通过对比分析两个案例，读者可以更好地理解[章节主题]在不同情境下的应用策略和实施要点。
```

#### 模块6：应用场景

```
## 六、应用场景

基于前面的理论分析，本节将系统介绍[章节主题]的各类应用场景。

在[场景1]中，[章节主题]主要用于[用途描述]。这种场景的特点是[特点描述]，需要特别注意[注意事项]。通过合理运用[章节主题]，能够有效解决[解决的问题]，实现[预期价值]。

在[场景2]下，[章节主题]则展现出不同的应用特点。这里的重点在于[重点内容]，关键在于[关键要素]。
```

#### 模块7：最佳实践

```
## 七、最佳实践

基于大量实践经验，我们总结出以下最佳实践：首先，在[实践环节1]中，应当遵循[原则或方法]，这是成功应用的基础。其次，在[实践环节2]时，需要重点关注[关注点]，这直接影响到最终效果。最后，通过[实践环节3]的持续改进，可以不断提升应用水平。
```

#### 模块8：对比分析

```
## 八、对比分析

[对不同的方案、观点或方法进行比较分析，以段落形式阐述各自的优缺点、适用场景和选择标准。]
```

#### 模块9：未来展望

```
## 九、未来展望

[探讨该主题的发展趋势、潜在挑战和未来方向。以叙述方式展开，避免使用要点列表。]
```

#### 模块10：本章小结

```
## 十、本章小结

通过对[章节主题]的系统学习，我们深入了解了其核心概念、实践应用和发展趋势。本章首先介绍了[核心概念1]，为理解整个主题奠定了基础；然后通过[核心概念2]的深入分析，揭示了主题的内在规律；最后结合实际案例，展示了理论知识的实践价值。

掌握[章节主题]不仅能够提升当前的[相关工作]能力，更为未来的[发展趋势]做好了准备。建议读者在实践中不断总结和反思，形成自己的理解和应用体系。
```

#### 模块11：延伸阅读

```
## 十一、延伸阅读

[推荐相关的学习资源、参考资料或扩展阅读材料，以段落形式介绍每个资源的特点和价值。]
```

## 输入内容

- 课程大纲：[生成的课程大纲]
- 分析素材：[整体分析结果和原文内容]
- 章节信息：[当前章节的标题和核心观点]
