# 专题文档撰写提示词

基于课程大纲和分析素材，生成专题详细内容。

## 撰写要求

1. **忠实原文**：保留文献中的核心观点、具体例子、重要表述
2. **叙述性写作**：以段落论述为主，避免用要点堆砌替代完整论证
3. **详尽完整**：每个概念都有完整的解释和论证，保持逻辑连贯
4. **避免外部扩展**：不添加文档外的知识和信息
5. **高保真正文增强**：
   - 不要把章节写成轻量摘要，应尽量保留原素材中有思想密度的观点、案例、判断、金句、工具选择理由、踩坑经验和实践细节
   - 关键案例应交代背景、问题、做法、结果和启发，不要压缩成一句"某案例说明了某道理"
   - 工具或方案取舍应说明选择理由、适用条件、局限和放弃其他方案的原因
   - 反直觉判断、个人经验、临时展开但有启发价值的观点，应自然融入正文
6. **自然表达**：
   - 禁止使用"原文中"、"根据原文"、"主讲人强调"、"主讲人指出"、"主讲人提到"、"主讲人认为"、"发言人表示"等表述
   - 禁止反复使用"现场演示"、"演示中"、"课程现场"、"现场问答"、"现场提到"、"本次分享中"等来源痕迹表述
   - 直接陈述内容本身，不强调内容来源归属
   - 将转录内容转换为客观的知识陈述形式
   - 去除发言人标记，将内容整合为连贯的叙述
   - 将操作型内容改写为"实践案例"、"应用场景"、"操作流程"、"一次典型迭代"等独立正文表达
7. **术语一致**：
   - 如已提供用户词典，保持大纲和素材中的校正结果
   - 统一使用词典中的正确术语写法，不恢复转录稿中的近似误写
   - 不新增词典外术语，也不对低置信内容做猜测式替换
   - 保留专有英文名称的原始写法、大小写、空格和连字符；`Course Generator`、`Claude Code`、`Codex`、`Cursor`、`Markdown`、`Git` 等不要翻译成中文
   - 可以解释专有名称的功能，但不要用中文译名替代英文名称；例如写 `Course Generator 用于...`，不要写"课程生成器用于..."
   - 文件名、命令名、API 名、Skill 名和项目名默认使用原文写法
8. **章节边界清晰**：
   - 严格围绕当前章节信息展开
   - 相近主题只在必要时建立连接，不重复其他章节的主体内容
   - 对核心章节给出更充分的方法论、框架和案例展开
9. **图片资产保真与正文配图克制**：
   - 如果课程大纲提供"本章正文配图"或图片资产表，只插入正文处理为`插入正文`的图片，不得遗漏这些正文配图
   - 使用图片资产表中的 `原始Markdown` 原样插入正文，包括 alt 文本、URL、扩展名和括号，不得改写、翻译、重命名、下载或重新上传
   - 同一章节内正文配图必须按 `IMG-xxx` 原始顺序出现，不要根据正文改写随意调换顺序
   - 图片应放在最相关的小节或段落之后；如果某张图片难以精确匹配，放在本章最接近的主题段落之后
   - 不要把所有正文配图集中堆到文末，除非该章没有更细的主题位置可放
   - 同一小节或同一位置连续图片原则上不超过 2 张，确有必要时最多 3 张；超过时保留最能说明问题的代表图，或分散到真正相关的不同段落之后
   - 不要为了保持独立正文而删除大纲已标注为`插入正文`的图片；这些图片是原始材料中的高价值结构线索
   - 如果大纲将图片标注为`仅资产表保留`或`跳过`，本章不插入该图
   - 插入图片前后可用一句短句说明它支撑的观点、流程或案例，不要写成"下面是 PPT"或"现场展示了这张图"
   - 图片数量减少不等于正文压缩；即使跳过连续操作页，也要保留其中有价值的操作逻辑、工具取舍、失败经验、限制条件和真实疑问
10. **问答自然融入**：
   - 原文中的 Q&A 是正文素材，必须按主题融入对应段落，不单独生成问答章节
   - 将问题转化为读者常见疑问、补充解释、案例回应或实践注意事项，将回答转化为正文论证
   - 保留问答中体现真实疑惑、实践阻力、误区和取舍的内容，过滤会务、设备调试等无课程价值的问答
   - 不机械保留"Q:"/"A:"格式，除非用户明确要求输出问答手册
   - 融入后不要写成"现场有人问"、"现场问答补充"，而应写成"实践中常见的疑问是..."或直接纳入对应论证
11. **重点表达**：
   - 允许使用少量 `### 1、关键判断`、`### 2、实践启发`、`### 3、方法框架` 等三级标题突出重点
   - 标题必须来自当前章节真实内容，不得为了模板完整硬造
   - 重点标题下仍以完整段落论述，不把章节退化成提纲
12. **标题与段落可读性**：
   - 所有三级标题必须使用 `### 1、标题` 格式，每个二级标题下从 `1` 重新编号
   - 段落目标长度约 160-240 个中文字符
   - 超过 260 字的段落应优先拆分，避免出现 300 字以上长段
   - 同一段只讲一个核心意思，不把概念定义、案例说明和价值判断塞进同一段

## 叙述性写作原则

**重要：以叙述性表达为主**

- 使用段落形式展开所有内容
- 每个概念都有完整的解释和论证
- 保持逻辑连贯，层层递进
- 避免使用"首先、其次、最后"等机械连接词
- 用自然流畅的叙述方式组织内容
- 在流程、步骤、术语映射、工具对比等场景，可以使用简短列表或箭头链路提升可读性，但不要把整章写成提纲
- 书稿素材取向下，宁可多保留一段有判断、有案例、有取舍的正文，也不要为了简洁删掉关键细节
- 图片是原始材料的结构线索，但正文只承载大纲筛选出的高价值配图；不要因为追求保真而把低价值、重复或会务性质图片塞进章节正文
- 克制插图只针对图片密度，不得把有价值正文同步压缩成摘要

## 章节模块库

根据内容特点，从以下模块中选择合适的组合：

### 可选模块

| 模块名称 | 适用场景 | 编号格式 |
|---------|---------|---------|
| 引言与概述 | 需要背景介绍和内容导航 | 一、 |
| 核心概念与原理 | 理论性内容，需要概念阐释 | 二、 |
| 理论框架 | 学术性、理论性较强的内容 | 三、 |
| 历史背景 | 需要时间线或发展历程的内容 | 四、 |
| 实践案例 | 有具体应用实例的内容 | 五、 |
| 应用场景 | 需要场景说明的内容 | 六、 |
| 最佳实践 | 需要操作指南的内容 | 七、 |
| 对比分析 | 需要比较不同方案/观点的内容 | 八、 |
| 未来展望 | 有前瞻性、趋势性内容 | 九、 |
| 本章小结 | 需要总结回顾的内容 | 十、 |
| 延伸阅读 | 有相关资源推荐的内容 | 十一、 |

### 模块选择原则

1. 根据内容类型选择合适的模块
2. 每章至少包含 2 个模块
3. 模块顺序遵循逻辑关系
4. 避免内容空洞的模块

### 默认推荐组合

- **理论类**：引言与概述 + 核心概念与原理 + 本章小结
- **实践类**：引言与概述 + 应用场景 + 最佳实践 + 本章小结
- **案例类**：引言与概述 + 实践案例 + 对比分析 + 本章小结
- **综合类**：根据内容特点灵活组合

### 模块详细说明

#### 模块1：引言与概述

```
## 一、引言与概述

本章将深入探讨[章节主题]的核心概念和实践应用。首先介绍该主题的背景和重要性，帮助读者建立整体认知框架。接着概述本章将要讨论的主要内容，为读者提供清晰的学习路径。
```

#### 模块2：核心概念与原理

```
## 二、核心概念与原理

[详细阐述核心概念，采用叙述性表达，每个概念都有完整的解释和论证。以段落形式展开说明，避免使用要点列表。整个论述过程要保持逻辑连贯，层层递进。]

### 1、[子主题1]

[以段落形式深入讲解，系统阐述该子概念的各个方面。首先介绍其基本定义和内涵，然后探讨其起源和发展历程，分析其理论基础和核心原理，最后说明其实际意义和应用价值。]

### 2、[子主题2]

[继续以叙述方式展开，与前一子主题形成逻辑递进关系。深入剖析该概念的本质特征，探讨其与相关概念的内在联系。]
```

#### 模块3：理论框架

```
## 三、理论框架

[阐述本主题的理论基础和框架结构，以段落形式说明核心理论的构成要素、内在逻辑和理论来源。]

### 1、[框架要素1]

[围绕一个框架要素展开，保持段落短而清楚。]
```

#### 模块4：历史背景

```
## 四、历史背景

[以时间线或发展历程的方式，介绍该主题的历史沿革、发展阶段和重要节点。以叙述方式展开，避免使用要点列表。]

### 1、[阶段或节点1]

[说明该阶段的背景、变化和影响。]
```

#### 模块5：实践案例

```
## 五、实践案例

通过具体案例来展示理论知识的实际应用，帮助读者将抽象概念转化为具体认知。

### 1、[案例名称]

这个案例生动地展示了[章节主题]在实际工作中的具体应用。案例背景是[背景描述]，面临的主要挑战是[挑战描述]。通过采用[具体方法]，成功实现了[预期效果]。整个过程中遇到的关键问题包括[问题清单]，而解决方案则体现了[章节主题]的核心价值。

### 2、[案例名称]

作为另一个典型应用场景，这个案例从不同角度展示了[章节主题]的灵活性和适用性。通过对比分析两个案例，读者可以更好地理解[章节主题]在不同情境下的应用策略和实施要点。
```

#### 模块6：应用场景

```
## 六、应用场景

基于前面的理论分析，本节将系统介绍[章节主题]的各类应用场景。

### 1、[场景名称]

在[场景1]中，[章节主题]主要用于[用途描述]。这种场景的特点是[特点描述]，需要特别注意[注意事项]。通过合理运用[章节主题]，能够有效解决[解决的问题]，实现[预期价值]。

### 2、[场景名称]

在[场景2]下，[章节主题]则展现出不同的应用特点。这里的重点在于[重点内容]，关键在于[关键要素]。
```

#### 模块7：最佳实践

```
## 七、最佳实践

基于大量实践经验，我们总结出以下最佳实践：首先，在[实践环节1]中，应当遵循[原则或方法]，这是成功应用的基础。其次，在[实践环节2]时，需要重点关注[关注点]，这直接影响到最终效果。最后，通过[实践环节3]的持续改进，可以不断提升应用水平。

### 1、[实践要点1]

[将具体实践写成短段，说明适用条件和操作要点。]
```

#### 模块8：对比分析

```
## 八、对比分析

[对不同的方案、观点或方法进行比较分析，以段落形式阐述各自的优缺点、适用场景和选择标准。]

### 1、[对比维度1]

[围绕一个维度比较不同方案，不混入其他维度。]
```

#### 模块9：未来展望

```
## 九、未来展望

[探讨该主题的发展趋势、潜在挑战和未来方向。以叙述方式展开，避免使用要点列表。]
```

#### 模块10：本章小结

```
## 十、本章小结

通过对[章节主题]的系统学习，我们深入了解了其核心概念、实践应用和发展趋势。本章首先介绍了[核心概念1]，为理解整个主题奠定了基础；然后通过[核心概念2]的深入分析，揭示了主题的内在规律；最后结合实际案例，展示了理论知识的实践价值。

掌握[章节主题]不仅能够提升当前的[相关工作]能力，更为未来的[发展趋势]做好了准备。建议读者在实践中不断总结和反思，形成自己的理解和应用体系。
```

#### 模块11：延伸阅读

```
## 十一、延伸阅读

[推荐相关的学习资源、参考资料或扩展阅读材料，以段落形式介绍每个资源的特点和价值。]
```

## 输入内容

- 课程大纲：[生成的课程大纲]
- 分析素材：[整体分析结果和原文内容]
- 章节信息：[当前章节的标题和核心观点]
- 高价值素材：[大纲中标注的本章需要保留的观点、案例、判断、金句、真实疑问或实践细节]
- 本章正文配图清单：[从图片资产表中筛选出的、正文处理为"插入正文"的本章图片，包含 IMG 编号、原始Markdown、邻近信息、图片价值、处理理由和放置建议；无则写"无"]
- 用户词典：[如果存在，列出 config/user_dictionary.yaml 中的 terms；如果不存在则写"未配置"]
