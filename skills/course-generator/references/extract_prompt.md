# 提取模式提示词

## 任务

根据客户的培训需求，从现有课程素材中提取并重组内容，生成定制化的课程方案文档。

## 输入

1. **客户需求描述**：包含受众画像、培训时长、重点方向等
2. **课程主题体系**：来自 `references/curriculum.md`，包含 6 阶段 18 讲的完整索引
3. **素材文件**：从知识库中定位的源文件内容

## 执行步骤

### 第一步：解析客户需求

从需求描述中提取关键信息：

- **受众**：企业法务 / 律师 / 法律研发 / 产品团队 / 混合
- **AI 基础水平**：零基础 / 有一定经验 / 技术背景
- **培训时长**：2-3 小时（一次性讲座）/ 半天 / 1 天 / 多天
- **重点方向**：认知普及 / 基础工具 / Agent/Skill / 实战案例 / 组织落地 / 超级个体
- **特殊需求**：特定业务场景、特定平台、特定行业

### 第二步：匹配课程讲次

根据解析的需求，在课程主题体系中匹配最合适的讲次组合：

1. 基于受众和基础水平，确定从哪些阶段中选取
2. 基于培训时长，确定选几讲（每讲约 60-90 分钟）
3. 基于重点方向，优先匹配对应讲次
4. 确保选取的讲次之间逻辑连贯

输出格式：
```
推荐讲次：第 X 讲、第 Y 讲、第 Z 讲
对应阶段：阶段 N
预估时长：约 X 小时
```

### 第三步：定位素材文件

根据匹配到的讲次，在知识库中定位对应的素材源文件：

- 对照 `curriculum.md` 中的"素材来源目录"，确定每个讲次的素材来源
- 使用 Glob 工具搜索知识库目录下的相关文件
- 优先读取各素材目录的总览文件（`00_*.md`）了解内容全貌
- 根据需要深入读取具体章节文件

### 第四步：提取与重组

从定位到的素材文件中提取相关内容，按以下原则重组：

1. **忠实原文**：保留素材中的核心观点、案例、数据，不添加素材外的信息
2. **逻辑重组**：按培训的时间线和认知逻辑重新组织，而非照搬素材的原始结构
3. **受众适配**：根据受众水平调整内容深度，基础受众侧重"是什么"和"怎么用"，技术受众侧重"为什么"和"怎么做"
4. **时长控制**：根据培训时长裁剪内容量，2-3 小时的讲座选取最精华的要点

### 第五步：输出课程方案

输出定制化的课程方案文档，格式如下：

```markdown
# [客户名称] 定制培训方案

> 受众：[受众画像]
> 时长：[预估时长]
> 生成日期：YYYY-MM-DD

## 培训目标

[基于客户需求，2-3 句话说明本次培训要达成的目标]

## 课程结构

### 第一部分：[主题]（约 XX 分钟）

**核心内容：**

1. [要点一]
2. [要点二]
3. [要点三]

**详细内容：**

[从素材中提取的详细内容，段落式展开]

### 第二部分：[主题]（约 XX 分钟）

[同上结构]

## 素材来源

- [素材文件路径 1]
- [素材文件路径 2]
```

## 质量标准

1. **针对性**：方案内容与客户需求高度匹配，不包含无关内容
2. **完整性**：在匹配的讲次范围内，核心要点不遗漏
3. **连贯性**：各部分之间逻辑衔接自然，不是简单拼接
4. **实用性**：包含具体案例、实操建议，不是空泛理论
5. **忠实性**：所有内容都有素材来源支撑，不凭空编造
