# 课程大纲生成提示词

请一次性分析以下所有文献内容，生成系统化的课程大纲。

## 核心要求

1. **整体把握**：理解所有内容的整体脉络和逻辑关系
2. **主题识别**：基于所有内容自动识别主要主题（不限于特定领域）
3. **内容优化**：
   - 识别并保留重要观点、具体例子、重要表述
   - 清洗冗余信息（如过度口语化表达、重复语句）
   - 去除发言人标记和内容来源归属表述（如"主讲人强调"、"主讲人指出"等）
   - 提取配图信息并合理安排位置
   - 将口语化内容转换为专业书面语，直接陈述内容而非强调说话者
4. **结构化组织**：按主题自动分组，标注各主题的核心观点
5. **完整提取**：保留关键观点、重要细节、具体案例、金句

## 案例融合原则

**重要：实践案例和展望内容的整合原则**

当分析内容包含实践案例、应用场景或未来展望时，应智能地将其融入相应的主题专题中，而不是创建独立的章节。

**整合策略：**
1. **案例匹配**：将案例内容按照主题相关性分配到对应的专题文档中
2. **内容融合**：在专题文档的"实践案例分析"部分自然地融入相关案例
3. **展望分配**：将未来发展趋势的内容融入相关主题的结尾部分
4. **智能判断**：AI 应根据内容的核心主题和应用领域进行合理分配

## 输出格式

```
===课程大纲开始===
课程标题：[基于原文生成的课程名称]

内容来源: [文件来源描述]
内容特征: [转录内容/普通文献/混合内容]

配图清单:
[如果有配图，列出图片链接和对应主题]

第一章 [主题名称] - 核心观点摘要
  [相关案例和应用场景将融入本章节]
第二章 [主题名称] - 核心观点摘要
  [相关案例和应用场景将融入本章节]
第三章 [主题名称] - 核心观点摘要
  [相关案例和应用场景将融入本章节]

输出计划：
- 总览文档：课程概览和学习路径
- 专题文档：每个主题的详细内容（包含融合的案例和展望）
===课程大纲结束===
```
