# 总览文档生成提示词

基于课程大纲，生成课程总览文档，帮助学习者快速了解课程全貌。

## 核心要求

1. **段落式写作**：以段落形式为主，避免把总览写成提纲堆砌
2. **全面覆盖**：涵盖课程概述、学习目标、课程结构、学习路径
3. **段落式展开**：每个部分都以段落形式详细阐述
4. **客观陈述**：禁止使用"主讲人强调"、"主讲人指出"、"主讲人提到"等表述，直接呈现课程内容本身
5. **避免套话**：根据课程实际内容组织语言，不使用固定的过渡词（如"首先...其次...最后..."），确保不同课程的总览各有特色
6. **术语一致**：如已提供用户词典，保持课程大纲中的校正结果，统一使用词典中的正确术语写法，不恢复转录稿中的近似误写
7. **结构导览可选**：
   - 不要固定生成"核心工作流地图"
   - 只有当课程大纲显示原文确实包含流程链路、方法框架、能力模型、系统架构或主题关系时，才加入简短结构导览
   - 导览标题应按内容自动命名，如"核心工作流地图""方法框架""能力模型""主题关系"
   - 没有明确结构时直接省略该部分，不要为了模板完整而硬加
   - 导览可以使用一行箭头链路或短列表，但必须服务于理解课程结构

## 输出格式

```
# [课程标题] - 总览

## 一、课程概述
[基于内容分析的简要概述，以段落形式展开介绍课程背景、核心价值和主要内容简介。每段聚焦一个核心意思，避免过长段落。]

[如果课程确实包含清晰的流程链路、方法框架、能力模型、系统架构或主题关系，可加入一个自动命名的结构导览部分；否则省略本部分。例如：
## 二、[核心工作流地图/方法框架/能力模型/主题关系]
[用一段话或一行箭头链路说明结构，例如：原始素材 -> 转录/OCR -> Markdown -> Skill -> Agent 编排 -> 知识库/交付物。标题和内容必须贴合素材，不固定套用该示例。]

后续二级标题序号根据是否加入结构导览自动顺延：如果没有结构导览，学习目标就是"二"；如果有结构导览，学习目标就是"三"。]

## [二或三]、学习目标
[根据课程实际内容，提炼 2-3 个核心学习目标，以段落形式阐述。目标应具体反映本课程的核心内容和预期收获，避免空泛表述。]

## [三或四]、课程结构
本课程共分为X个章节，每个章节聚焦一个核心主题。

[以段落形式逐一介绍各章主题和核心内容，注意：
- 根据各章实际内容描述其侧重点，不使用固定句式
- 突出章节之间的逻辑递进关系
- 说明核心章节及其原因
- 用自己的语言组织，避免模板化表达]

## [四或五]、学习路径建议
[根据课程的实际内容结构和难度梯度，给出具体的学习建议。建议应体现课程内容的特点，而非通用套话。]
```

## 输入内容

- 课程大纲：[生成的课程大纲]
- 课程标题：[AI 生成的课程名称]
- 用户词典：[如果存在，列出 config/user_dictionary.yaml 中的 terms；如果不存在则写"未配置"]
