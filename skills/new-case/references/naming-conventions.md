# 案件编号与命名规范

## 目录占位符

本文档和模板中的 `{00}`~`{11}` 为目录占位符，对应预设配置中的 `directories` 定义。生成文件时替换为配置的完整目录名（含编号和图标），如 `{00}` → `00 - 📅 日程管理`。

## 案件/项目编号

### 诉讼案件

格式：`{YY}{MM}{序号} {原告简称} 与 {被告简称} {案由关键词}`

| 要素 | 规则 | 示例 |
|------|------|------|
| 日期 | YYMM，同月案件序号递增 | `260221`、`260302` |
| 原告简称 | 取姓名或公司核心名称 | `张家宁`、`甲公司` |
| 被告简称 | 同上 | `pokebone 模型`、`乙公司` |
| 案由关键词 | 取最核心的案由词 | `著作权`、`合同纠纷` |

示例：
- `260221 张家宁 与 pokebone 模型 著作权`
- `260315 甲公司 与 乙公司 合同纠纷`
- `260410 张三 与 李四 人身损害赔偿`

**多原告/多被告**：取主要当事人简称，用"等"字替代其余。
- `260501 张家宁等 与 pokebone等 著作权`

### 咨询项目

格式：`{YYMMDD} {客户/对方简称} {事项关键词}`

| 要素 | 规则 | 示例 |
|------|------|------|
| 日期 | YYMMDD，首次咨询日期 | `260323`、`260410` |
| 客户简称 | 取联系人或公司名 | `张美金`、`闪电自行车` |
| 对方/事项 | 对方当事人或事项关键词 | `浙江腾宇包装公司`、`普洛姆` |
| 事项关键词 | 概括咨询内容 | `咨询`、`商业秘密`、`外观专利` |

示例：
- `260323 张美金 浙江腾宇包装公司 咨询`
- `260404 顾忆芬 苏州物睿科技 商业秘密`
- `260410 闪电自行车配件 普洛姆 外观专利`

### 商标项目

格式：`{YYMMDD} {客户简称} {业务类型}`

| 要素 | 规则 | 示例 |
|------|------|------|
| 日期 | YYMMDD，首次接洽日期 | `251119`、`260104` |
| 客户简称 | 取公司核心名称 | `方帅体育`、`东莞伊甸文化` |
| 业务类型 | 业务子类型关键词 | `商标注册`、`商标异议`、`商标无效`、`商标复审`、`商标变更` |

业务子类型枚举：

| 业务类型 | 触发场景 |
|----------|----------|
| 商标注册 | 首次申请商标 |
| 商标异议 | 对初审公告的他人商标提出异议 |
| 商标复审 / 驳回复审 | 对驳回决定申请复审 |
| 商标无效 | 对已注册/公告商标提出无效宣告 |
| 商标变更 | 商标变更、转让、续展 |

示例：
- `251119 方帅体育 商标注册`
- `260104 东莞伊甸文化 商标注册`
- `260203 雅艺轩 商标异议`
- `240304 米世达 商标无效`
- `220411 潽洛因思 商标复审`

**注意**：
- 一个项目可包含多个商标/申请号（如方帅体育的 88753562 + 90922527），但项目名不重复列商标名/申请号——具体清单进入项目信息卡的"关联申请号"表
- 商标名（如"镭战联盟"）可省略，因为它和"商标注册"组合后，名称会过长
- 如果一个客户一次性委托多个独立业务（如同时注册 + 异议 + 复审），建立 1 个项目（项目信息卡的"关联申请号"表追踪所有申请号），目录结构仍是 7 个平铺分类

### 专利项目

格式：`{YYMMDD} {客户简称} {业务子类型}`

| 要素 | 规则 | 示例 |
|------|------|------|
| 日期 | YYMMDD，首次接洽日期 | `260316`、`250325` |
| 客户简称 | 取公司核心名称 | `上海生机智联`、`百林科` |
| 业务子类型 | 9 个专利业务子类型之一 | `发明专利申请`、`专利无效`、`专利检索` |

业务子类型枚举（9 个）：

| 业务子类型 | 触发场景 |
|-----------|----------|
| 发明专利申请 | 发明专利首次申请（保护期 20 年，需实质审查）|
| 实用新型申请 | 实用新型申请（保护期 10 年，形式审查）|
| 外观设计申请 | 外观设计申请（保护期 15 年，形式审查）|
| 专利无效 | 对已授权专利提出无效宣告请求 |
| 专利复审 | 对驳回决定申请复审 |
| 专利检索 | 专利数据库检索、可专利性分析、侵权风险分析 |
| PCT国际申请 | PCT 国际专利申请 |
| 专利变更 | 著录项目变更、专利权转让、许可备案 |
| 年费管理 | 专利年费缴纳追踪、滞纳金监控 |

示例：
- `260316 上海生机智联 发明专利申请`
- `250325 百林科 专利无效`
- `220331 金童 专利无效`
- `211221 无锡亿富 专利检索`
- `210513 晟杰机器人 发明专利申请`

**注意**：
- 一个项目可包含多个专利/申请号（如同一客户的多件发明），项目名不重复列专利名/申请号
- 专利名（如"智能体"）可省略，避免项目名过长
- 业务子类型明确区分（发明 vs 实用新型 vs 外观），便于法定期限追踪
- 复审/无效/检索类项目命名简化（"专利无效"而非"发明专利无效宣告请求"）

## 案件名称（仅诉讼案件）

格式：`{原告}+等+诉+{被告}+等+{案由}+案`

示例：
- `张家宁 诉 pokebone 模型 著作权侵权案`
- `甲公司 诉 乙公司 合同纠纷案`
- `张三等 诉 李四等 人身损害赔偿案`

**注意**：案件名称用于法律文书标题和案件信息看板，与案件编号是两个不同的概念。

## 材料编号：参照度分级

### 格式

`{参照度}-{三位序号} {描述}.{扩展名}`

### 参照度级别

| 级别 | 含义 | 适用材料 |
|------|------|----------|
| A | 活文档 | 律师持续迭代的分析、方案、测算、策略建议、事实梳理 |
| B | 常查 | 关键录音转写、核心截图、订单数据、合同原件 |
| C | 存档 | AI 咨询文档、重复截图、低信息量材料、已归档的中间版本 |
| D | 待激活 | 压缩包原件、未处理原始文件、转化诉讼时可能需要的材料 |

### 规则

- **序号全局唯一不重叠**：A-001、A-002、B-003、C-004...，从最大序号即可看出材料总量
- **A 级内部排序原则**：一手沟通记录 → 案件分析 → 策略方案。一手信息（律师-客户咨询录音、当事人陈述等）编号最前，分析类其次，策略/方案类最后
- **参照度可调整**：随项目进展可升降级（如 B→A、C→B），调整时更新文件名和材料清单
- **初始赋值**：
  - legal-material-organizer 扫描时默认编为 D（待评估）
  - 完成内容理解后按材料类型和内容判定参照度
  - 律师手动创建的文档直接编为 A

### 命名示例

```
A-001 律师-客户初次咨询录音转写-5月19日.md    ← 一手沟通记录，A 级最前
A-002 案件事实与争议焦点.md                    ← 案件分析类
A-005 甲方诉讼主体资格法律研究.md              ← 法律研究类
A-010 赔偿金额分项测算.md                      ← 分析测算类
A-014 法律服务方案.md                          ← 策略方案类，A 级最后
B-009 小鹅通订单数据-2025.06至2026.03.csv
B-015 聊天截图-文案起草授权-大胆点.jpg
C-019 追问1-担心道歉被用作起诉证据.docx
D-008 师白-链路.zip
```

## 文件命名规则

### 管理文件

| 文件类型 | 诉讼命名 | 咨询命名 | 商标命名 | 专利命名 |
|----------|----------|----------|----------|----------|
| 案件/项目信息 | `[案件编号] 案件信息.md` | `[项目编号] 项目信息.md` | `[项目编号] 项目信息.md` | `[项目编号] 项目信息.md` |
| 工时记录 | `[案件编号] 工时记录.md` | 不生成 | 不生成 | 不生成 |
| 期限管理 | `[案件编号].yaml` | 不生成 | 不生成（续展提醒进待办）| 不生成（年费提醒进待办）|
| 待办事项 | `[案件编号] 待办事项.md` | `[项目编号] 待办事项.md` | `[项目编号] 待办事项.md` | `[项目编号] 待办事项.md` |

### 其他文件

- 使用中文描述性命名
- 日期前缀格式：`YYMMDD `（如 `260323 电话咨询.md`）
- 录音文件保留原始扩展名（.mp3/.aac/.wav）
- 转写文稿与录音同名，扩展名改为 `.md` 或 `.txt`
- 法律服务方案：`YYMMDD [方案名称]法律服务方案.docx`（放在工作文件目录内）

## 向后兼容

旧格式 `M-XXX` 可在新系统中保留为映射关系，无需立即全部重命名。新建项目统一使用参照度分级编号。
