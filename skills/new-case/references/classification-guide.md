# 材料分类详细指南

本文档补充 `assets/*.yaml` 中 `classification` 关键词匹配之外的分类决策逻辑。

## 分类原则

1. **顺序匹配**：按 YAML 中 `classification` 列表顺序，命中第一条即停止
2. **文件名优先**：匹配对象为文件名（含扩展名），不区分大小写
3. **内容辅助**：文件名无法判断时，可读取文件内容前几行辅助判断
4. **保留时间戳**：复制/移动文件必须用 `cp -p`（或 `mv`），**禁止**用裸 `cp`。律师按文件 mtime 判断合同签订和项目推进时间，破坏时间戳会导致历史追溯失真。
5. **保留用户自建子目录**：客户/律师在原项目中自建的次级子目录（如 `NOVA ARTS/`）原样移入新位置的对应分类目录，**禁止**拉平/抽取子目录内容到父目录。例如 `02 商标图样/NOVA ARTS/` 整个子目录应该跟着 `02 商标图样/` 一起移动，子目录内 6 个文件保持原位。例外：当 4+ 申请号文件名冲突时，再考虑建按申请号分子目录。

## 模糊场景处理

### 录音归属：律师-客户 vs 客户-对方

- **律师与客户的沟通录音** → 01-客户材料（属于客户提供的咨询记录）
- **客户与对方的交涉录音** → 01-客户材料（客户提供的事实材料）
- 录音来源无法判断时，默认归入 01-客户材料

### 法律服务方案

法律服务方案统一放在 **02-工作文件**，不放在根目录。方案属于律师产出，且经常迭代更新。

### 视频文件归类

- 客户与对方沟通的现场录像 → 01-客户材料
- 客户拍摄的证据固定视频 → 01-客户材料
- 律师与客户的咨询录像 → 01-客户材料

### 转化后的子案件文件夹

咨询项目转化为诉讼案件后，可能产生以法院案号命名的子文件夹。处理方式：
- 保留在咨询项目根目录下，不归类到 01/02
- 在项目信息文件的"关联案件"部分记录子案件编号和路径
- 子文件夹内部结构遵循 litigation 预设

### 录音与转写文稿配对

转写文稿与对应录音必须放在**同一目录**。如果录音在 01-客户材料，转写也放 01。

### 无法判断的文件

归入最相关的目录：
- 咨询项目 → `01 - 📂 客户材料`（默认归入客户材料）

## 商标项目分类（按业务子类型选 2 个模板）

### 核心设计

商标项目**不**使用单一套 7 目录超集——v1.3.0 把 5 证据材料等无效/异议类目录暴露在注册类项目里，造成空目录，违反"不创建空目录"原则。

**改为单一预设下挂 2 个业务子模板**，按业务子类型自动选模板。**目录顺序按"项目进展"排列**（DEC-008 + DEC-009）：
- 注册模板：1 委托 → 2 图样 → 3 申请书 → 4 律师工作 → 5 官方文书 → 6 商标注册证（独立） → 7 发票财务（7 目录）
- 异议/复审/无效模板：1 委托 → 2 图样 → 3 申请书 → 4 律师工作 → 5 证据材料 → 6 官方文书 → 7 商标注册证（独立） → 8 发票财务（8 目录）

| 模板 | 业务子类型 | 目录数 | 关键特征 |
|------|----------|--------|----------|
| **注册模板** (registration) | 注册/变更/转让 | **7** | 无 5 证据材料（注册阶段无对抗）；含 6 商标注册证（独立，DEC-2026-06-11）；**工作记录/沟通并入 1 委托**（DEC-009）|
| **异议/复审/无效模板** (opposition) | 异议/复审/无效 | **8** | 含 5 证据材料（对抗性案件核心）+ 7 商标注册证（独立，DEC-2026-06-11）；**工作记录/沟通并入 1 委托**（DEC-009）|

### 模板 1: 商标注册模板（7 目录）

适用于商标注册、商标变更、商标转让。

```
项目根/
├── 01 - 📑 委托材料/         # 委托书/合同/协议/执照/身份 + 工作记录/沟通
├── 02 - 🖼️ 商标图样/         # 商标图样（彩色稿/黑白稿/源文件 .ai/.psd/画板）
├── 03 - ✍️ 申请书/            # 律师起草（注册/变更/转让申请书）
├── 04 - 📊 律师工作/          # 检索/分析/驳回应对/复审应对/参考材料 + 进程跟踪
├── 05 - 🏛️ 官方文书/          # 商标局文书（受理/补正/缴费/驳回/复审决定/变更证明；不含注册证）
├── 06 - 📜 商标注册证/         # 商标注册证 PDF（独立一级目录；YYMMDD 日期前缀）
└── 07 - 💰 发票财务/           # 律师费发票、官费发票

**vs 早期设计**：
- v1.3.0 7 目录超集 → 7 目录（去掉 5 证据 + 新增 6 商标注册证独立一级目录）
- 1 委托材料 含 沟通/工作记录（DEC-009 合并）
- 4 律师工作 与 文件进程 合并

### 模板 2: 商标异议/复审/无效模板（8 目录）

适用于商标异议、商标复审、商标无效。

```

### 模糊场景处理

#### 多申请号的同类文件

- 多个申请号的申请书 → 都进 `03 - ✍️ 申请书/`，靠文件名识别
- 多个申请号的官方文书 → 都进 `05 - 🏛️ 官方文书/`（注册模板）或 `06 - 🏛️ 官方文书/`（异议模板）
- 商标注册证（PDF）→ 进 `06 - 📜 商标注册证/`（注册模板）或 `07 - 📜 商标注册证/`（异议模板），**不在 05/06 官方文书**；加 YYMMDD 日期前缀
- 委托书（覆盖多申请号时）→ 进 `01 - 📑 委托材料/`，**不分拆**
- 商标局缴费通知书 → 进官方文书目录；律师费发票 → 进 `发票/财务/`（实际目录名 `💰 发票财务`，因 macOS 不支持文件名含 `/`）

#### "驳回通知书" vs "驳回复审申请书"

两者**不**放在同一目录：
- 驳回通知书（商标局发的）→ 官方文书
- 驳回复审申请书（律师起草的）→ 申请书

#### 委托书版本

委托书有多版本（.doc / .jpg / .pdf / 盖章版）—— 全部放 `01 - 📑 委托材料/`，**不**分散。

#### 引证商标材料

异议/复审/无效案件中引证他人商标的材料：
- 引证商标号、申请人、注册证复印件 → `05 - 📎 证据材料/`
- 引证商标的查询分析 → `04 - 📊 律师工作/`

#### 商标监测报告

定期对客户商标做状态监测的报告 → `04 - 📊 律师工作/`，文件名带日期便于追踪多次监测。

#### 历史参考案件材料

客户提供的**历史参考件**（如某商标之前的驳回复审材料）—— 不属于当前在办项目，按文件类型归到对应目录即可。项目信息卡的"关联案件"小节注明来源案件号。

### 官方文书 PDF 自动加发文日期前缀

**详见 SKILL.md "第三步半"**。要点：

- 扫描 `04 - 🏛️ 官方文书/`（及 `03/05` 中官方文书类 PDF）
- 跳过已带 `YYMMDD ` 前缀的文件
- 用 `pdftotext -l 1` 提取首页文本（文字层）或 `mineru-ocr`/`legal-ocr` 跑 OCR（扫描件）
- 优先匹配"发文日期" / "决定日期" / "本决定书" / "签发" 附近的日期
- 兜底：取 PDF 全文最晚的合理日期（2020 年后）
- `mv` 改名（保留 mtime），格式 `YYMMDD 原文件名.pdf`
- 更新 `trademark-info.md` 的"材料清单"和"关键时间线"

**日期格式匹配优先级**：
1. 阿拉伯数字：`2025年07月10日` / `2025-7-10` / `2025/7/10` / `2025.7.10`
2. 中文数字：`二〇二五年七月十日` / `二○二五年七月十日`
3. 6 位纯数字：`20250710`（少见）
4. PDF CreationDate 元数据（最后兜底）

## 专利项目分类（按业务子类型选 3 个模板）

### 核心设计

专利项目**不**使用单一套 11 目录超集——早期设计（v1.3.1）把 8 证据/9 对方提交等无效案件目录暴露在申请类项目里，造成空目录，违反"不创建空目录"原则。

**改为单一预设下挂 3 个业务子模板**，按业务子类型自动选模板。**目录顺序按"项目进展"排列**（DEC-008 + DEC-009）：
1. 委托材料（含工作记录：沟通录音/邮件/会议纪要）→ 2. 申请清单/客户提供 → 3. 客户提供/律师工作 → 4-5. 律师工作/申请书 → 6-7. 国知局文件 → 7-8. 发票/财务

| 模板 | 业务子类型 | 目录数 | 关键特征 |
|------|----------|--------|----------|
| **申请模板** (application) | 发明申请/实用新型/外观/PCT | **7** | 无 证据/对方提交（申请阶段无对抗）；律师工作**与文件进程合并**；**工作记录并入 1 委托材料** |
| **复审/无效模板** (invalidation) | 复审/无效 | **8** | 含 证据材料/对方提交；无 申请清单（不是新申请）；律师工作与文件进程合并；**工作记录并入 1 委托材料** |
| **检索/其他模板** (search) | 检索/布局/变更/年费 | **4** | 轻量结构；任务清单为"拟检索/拟变更/代缴年费"；**工作记录并入 1 委托材料** |

### 模板 1: 申请模板（7 目录）

适用于发明专利申请、实用新型申请、外观设计申请、PCT 国际申请。

```

├── 3 - 📥 客户提供/         # 技术交底书、现有技术、产品说明书、图纸
├── 4 - 📊 律师工作/         # 检索报告/新颖性创造性分析 + 文件进程节点（合并）
├── 5 - ✍️ 申请书/            # 律师起草（请求书/权利要求/实审请求/PCT）
├── 6 - 🏛️ 国知局文件/         # 受理/缴费/审查意见/驳回/授权/证书
└── 7 - 💰 发票/财务/         # 代理费发票、官费
```

**vs 早期设计**：
- v1.3.1 11 目录超集 → v1.3.2 8 目录 → **v1.3.4 7 目录**（去掉 8 证据/9 对方提交 + 合并 文件进程 + 合并 工作记录入 1 委托材料）

### 模糊场景处理

#### 多申请号的同类文件

- 多个申请号的申请书 → 都进 `5 - ✍️ 申请书/`（申请模板），靠文件名识别
- 多个申请号的国知局文件 → 都进 `6 - 🏛️ 国知局文件/`（申请模板）或 `7 - 🏛️ 国知局文件/`（复审/无效模板）
- 委托书（覆盖多申请号时）→ 进 `1 - 📑 委托材料/`，**不分拆**

#### 复审/无效案件的对比文件

- 复审/无效模板：`5 - 📎 证据材料/` 下保留用户自建的"对比文件 1/D1"等子目录
- **不**把对比文件拉平到 `5 - 📎 证据材料/` 根目录
- 例：`5 证据材料/对比文件 1/`、`5 证据材料/D2/` 整体移入

#### 检索类 vs 申请类

检索类项目（search 模板）只需要：
- `1 委托材料`（检索委托协议）
- `2 沟通记录`（检索需求沟通）
- `3 任务清单`（拟检索目标）
- `4 律师工作文件`（检索报告）

## 诉讼 vs 咨询 vs 商标 vs 专利分类差异
