# 高级案例套路

本文件用于复杂案件和高阶论证图。`scene-library.md` 解决“选什么场景”，`scene-composition-playbook.md` 解决“基础怎么画”，本文件解决“如何把复杂材料压缩成有说服力的主图”。

## 适用条件

遇到以下情形时读取本文件：

- 案件材料跨越多年、多主体、多法律关系，一张普通关系图无法承载。
- 争点不是单一事实，而是不同制度路径、不同法律性质或不同处理结果的比较。
- 需要把市场背景、政策变化、工程进度、票据回路、股权变动等外部线索和案内事实叠加。
- 已有事实很多，但图表真正要表达的是责任归属、损失范围、主观意图或因果关系。

## 通用拆解顺序

1. **先做细节图**：按合同、资金、票据、股权、工程节点、工商登记、程序节点分别拆开。
2. **找连接点**：优先找共同主体、共同账户、同一票据、同一项目、同一合同、同一时间节点。
3. **确定主线**：如果争点是取得时间、迟延期间或期限，选时间主线；如果争点是谁对谁承担权利义务，选法律关系主线；如果争点是不同规则导致不同结果，选制度路径主线。
4. **先组合后删减**：组合总图后，用 3S 原则删去无关支线，保留能直接支撑主观点的节点。
5. **主图加附图**：主图只回答一个问题，明细台账、证据目录、分支关系放附图。

## 七类高阶套路

### 1. 工期延误责任图

适用：多楼栋、多工序、多延误原因的建设工程案件。

编排方法：

- 先按楼栋或施工模块拆成若干时间轴。
- 再抽取共同延误逻辑，形成主图。
- 上半部分放计划进度，下半部分放实际进度。
- 关键延误区间用强调色，责任主体用分层或色块区分。
- 如果某一工序构成主要延误原因，单独放大该工序，标出延误天数和占比。

避免事项：

- 不把每个施工节点都塞进主图。
- 不只画总工期，必须标出关键工序、责任主体和证据来源。

### 2. 制度路径对比图

适用：同一事实在两种法律规则、政策路径或处理机制下产生不同结果。

编排方法：

- 左右两栏分别放路径 A 与路径 B。
- 起点保持一致，末端显示权利、资金、责任或程序结果差异。
- 中间节点只保留导致结果分叉的关键规则。
- 结果栏用同一指标表达，便于比较。

避免事项：

- 不把法律论证全文写进流程节点。
- 不把路径差异画成事实差异；事实相同、规则不同要明确分层。

### 3. 背景趋势叠加图

适用：当事人行为受市场、政策、价格、行业周期影响，需要证明行为动机或商业合理性。

编排方法：

- 上层用折线图表达背景趋势，如价格指数、材料价格、市场景气、监管节奏。
- 下层用时间轴表达案内关键行为。
- 用竖向参考线把背景变化和案内行为对齐。
- 结论栏写明背景变化如何解释行为选择。

避免事项：

- 外部数据必须标注来源和口径。
- 背景趋势只能辅助解释，不能替代证据证明。

### 4. 两张票据/两条资金路径对比图

适用：票据案件中存在多张票据、返还回路、损失抵扣或责任范围区分。

编排方法：

- 将两张票据拆成上下两条路径或左右两栏。
- 票面记载、实际流转、资金流向分别使用不同线型。
- 返还或抵扣款项只连接到对应票据，不跨线混用。
- 右侧统一计算损失差额和责任范围。

避免事项：

- 不把票面流转和实际流转混成一条线。
- 不让返还路径同时指向多个损失来源。

### 5. 真实意思/行为动机图

适用：需要透过表面行为解释当事人真实目的、商业动机或履约态度。

编排方法：

- 底层仍以可核对事实为主，不直接画心理推测。
- 将背景趋势、合同履行节点、关键函件、资金投入/撤出等行为对齐。
- 用“可能解释/推断依据”侧栏表达观点，不把推断写成确定事实。
- 对方解释和本方解释可左右对比。

避免事项：

- 不用图表直接断言主观状态。
- 不把公开背景数据和案内证据混为同一证明等级。

### 6. 长周期房地产复合关系图

适用：合作开发、项目置换、股权变更、工商变更、第三方接盘同时出现。

编排方法：

- 先拆三张细节图：合作关系、项目权益流转、股权/工商变更。
- 用核心项目和核心主体作为连接点。
- 不同法律关系使用不同分区和颜色。
- 主图只展示影响合同是否继续履行、主体资格和权益归属的关系。

避免事项：

- 不把每一次工商变更都画进主图。
- 不把项目空间争议和资金/股权关系混在同一平面图里。

### 7. 图表化证据目录

适用：图中每个节点都需要直接支撑证据，或案件需要用图表重排证据目录。

编排方法：

- 给主图节点编号，如 F01、F02、R01。
- 附图或侧栏列证据编号、页码、证明目的、争议状态。
- 对证据缺口使用灰色或“待补强”标注。
- 主图和证据目录保持一一对应，避免读者在卷宗中反复寻找。

避免事项：

- 不在主图主体节点内塞长证据摘录。
- 不把无证据支撑的事实画成确定事实。

## 选择建议

| 任务 | 优先套路 |
|------|------|
| 多楼栋或多工序延误 | 工期延误责任图 |
| 不同法律路径导致不同结果 | 制度路径对比图 |
| 市场或政策背景解释履约行为 | 背景趋势叠加图 |
| 票据损失范围、返还抵扣 | 两张票据/两条资金路径对比图 |
| 行为表象与商业动机不一致 | 真实意思/行为动机图 |
| 房地产合作开发长周期复合案 | 长周期房地产复合关系图 |
| 证据多且需要直接支撑图中节点 | 图表化证据目录 |

## 输出要求

- 高阶图仍遵守“一图一观点”：主图必须能用一句话概括。
- 主图最多承载 2 类信息形态；第 3 类信息应放附图或侧栏。
- 所有外部背景、推断、争议事实都要降低证明等级并显式标注。
- 使用模板时优先从 `templates/` 中复制 `.drawio`，再替换文字、节点和线条。
