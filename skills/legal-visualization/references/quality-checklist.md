# 质量检查清单

出图前后均使用本清单。不能通过的项目，要先修正再交付；确实无法验证的项目，应在最终说明中写明。

## 事实与法律表达

- [ ] 图表只使用材料中已有事实，或明确标注“待补充/待核/一方主张”。
- [ ] 争议事实没有被画成确定事实。
- [ ] 法律结论与事实节点分开，结论放标题、副标题、右上角结论栏或说明栏。
- [ ] 给法官的图保持客观克制；给客户的图可以突出策略和风险。
- [ ] 每张图只有一个核心观点。

完整硬约束（4 条）见 `SKILL.md` 顶部"硬约束"段。

## 场景选择

- [ ] 已按 `scene-routing-guide.md` 先缩小候选场景，再按 `scene-library.md` 选择场景 ID。
- [ ] VizSpec 中已记录主场景、备选场景和未选原因。
- [ ] 图表类型与核心争点匹配：时间、关系、数据、流程、空间或证据。
- [ ] 复杂案件已先拆细节图，再合成主图。
- [ ] 不把三类以上信息强行塞入一张图。

## 版面与视觉

- [ ] 同一主体使用同一颜色。
- [ ] 同一类型关系使用同一线型。
- [ ] 强调色有明确含义，不超过 3 个主要强调色。
- [ ] 线条没有明显交叉；没有长线条绕行。
- [ ] 连线标签没有互相重叠，没有压住箭头或节点边界；长标签已移到独立文本块、侧栏或图例。
- [ ] 连接多的主体位于中心或靠近相关节点。
- [ ] 图表主体文字短；长说明放脚注、侧栏或附表。
- [ ] 节点对齐，间距充足，图面不拥挤。
- [ ] 有必要的标题、图例、注释和证据来源。

## draw.io XML

- [ ] XML 包含 `mxGraphModel`、`root`、`id="0"`、`id="1"`。
- [ ] 每个 edge 都有 `mxGeometry relative="1"` 子元素。
- [ ] 所有顶层元素 `parent="1"`，容器内元素使用相对坐标。
- [ ] 节点有明确宽高，中文不被截断。
- [ ] 注释中没有可能破坏 XML 的字符组合。

自动化自检脚本见 `scripts/validate_drawio.py`，可作为质量门禁的预检环节。

## 导出文件

- [ ] 已保存 `.drawio` 源文件。
- [ ] 已导出 SVG 或 PNG；用户要求时导出 PDF。
- [ ] 图片打开后非空白。
- [ ] 画布边界没有裁切主体或图例。
- [ ] 文字、箭头、虚线、颜色在导出图片中可见。
- [ ] 文件名能反映案件或图表用途。

## 交付前说明

- [ ] 说明输出文件路径。
- [ ] 说明采用的场景和主图逻辑。
- [ ] 说明未能验证或无法导出的环节。
- [ ] 提醒用户核对未脱敏材料和隐私信息。
