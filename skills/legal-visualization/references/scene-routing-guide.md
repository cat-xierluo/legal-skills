# 场景路由规则

本文件用于在大场景库中做第一轮筛选。先用本文件选出 1-3 个候选场景，再读取 `scene-library.md` 中对应章节，最后按 `scene-composition-playbook.md` 编排图面。

## 路由顺序

1. **先看用户明确指定**：用户点名“时间轴、法律关系图、服务路线图、证据矩阵、交易架构图”等图形时，优先尊重指定图形；只有明显不适合材料时才调整。
2. **再看受众**：法官/仲裁庭优先案件事实、争点、证据、法律关系；客户优先路径、风险、方案、成本、配合事项；业务部门优先流程、责任、交付物；律师团队优先任务、证据、文书、复核。
3. **再看任务动词**：说明、汇报、选择、推进、证明、反驳、审查、整改、交付、复盘分别指向不同场景组。
4. **再看材料阶段**：咨询/报价/委托、办案/庭审/执行、证据/文书、交易/合规、归档/复盘分别进入不同章节。
5. **最后看信息形态**：时间、关系、流程、数据、证据、空间、决策、看板决定具体图表形态。

## 评分法

候选场景按 10 分制快速评分：

| 维度 | 分值 | 判断 |
|------|------|------|
| 受众匹配 | 3 | 输出对象需要这张图吗 |
| 任务动词匹配 | 3 | 用户到底要决策、说明、证明、推进还是管理 |
| 领域名词匹配 | 2 | 材料是否明确属于该业务领域 |
| 信息形态匹配 | 1 | 时间、关系、流程、证据、数据等是否匹配 |
| 用户指定格式 | 1 | 用户是否明确点名图形或交付格式 |

扣分规则：
- 场景需要材料中没有的关键事实：-2。
- 场景过于泛化，无法突出主问题：-2。
- 场景会把客户沟通图画成律师内部流程，或反向混淆：-2。
- 场景会把争议事实画成确定事实：-2。

若最高分并列，优先级为：用户指定 > 受众匹配 > 更窄业务领域 > 更接近当前材料阶段 > 通用场景。

## 冲突处理

| 冲突 | 优先规则 |
|------|------|
| SP 服务方案 vs CX 客户生命周期 | 报价、服务价值、服务范围选 SP；咨询、委托、材料配合、进度、结案续约选 CX |
| LP 案件办理 vs RP 争议路径 | 单一案件从接案到执行选 LP；多条解决路径比较、谈判/仲裁/诉讼选择选 RP |
| LP 案件办理 vs EV 证据工作 | 案件推进主线选 LP；证据发现、固定、补强、质证作为主问题选 EV |
| EV 证据工作 vs G-14/G-15/E 证据图 | 证据工作过程选 EV；提交给法庭证明争点选 G-14/G-15；复杂图表组合或证据目录联动选 E |
| WD 文书生产 vs LP 案件办理 | 生成、复核、版本管理某份文书选 WD；案件阶段推进选 LP |
| WD 文书生产 vs CT 合同流程 | 合同审查意见的生产和交付选 WD；企业合同审查流程、履约管理选 CT |
| CT 合同流程 vs MA 交易项目 | 合同管理和履约闭环选 CT；交易架构、交割条件、尽调问题选 MA |
| CG 合规治理 vs BC 企业全周期 | 单一合规问题、整改、审批矩阵选 CG；企业阶段、年度顾问、业务线全景选 BC |
| CG 合规治理 vs HR/IP/RG 专题 | 劳动、数据/IP、监管已有明确专题时选 HR/IP/RG；跨部门通用合规选 CG |
| CX 客户生命周期 vs TM 团队协作 | 给客户看配合事项和进度选 CX；给团队看负责人、截止日、复核选 TM |
| DR 债务化解 vs LP/RP 执行路径 | 债务结构、清偿顺位、财产线索选 DR；胜诉执行推进选 LP；执行转破产路径选择选 RP |
| G 通用图表 vs 业务专题 | 专题场景能命中时不用 G；G 只作为基础图形或没有明确业务类型时使用 |

## 主图与附图

- 只选一张主图：当用户要求“一步到位”“一张图”“给客户看”时，主图必须回答一个问题。
- 主图 + 附图：当材料同时涉及关系、时间、证据、金额、流程时，主图只保留核心路径，附图承接细节。
- 不要为了覆盖场景库而多出图；多图只在读者需要不同层级信息时使用。

## 输出路由说明

生成 VizSpec 前，先写一个简短路由结论：

```yaml
routing:
  audience: "客户/法官/业务团队/律师团队"
  task: "说明/证明/选择/推进/管理/交付"
  primary_scene: "场景 ID - 场景名"
  alternatives_considered:
    - "场景 ID - 未选原因"
  reason: "一句话说明为什么选主图"
```

如果用户只要最终图片，不需要把路由结论展示给用户；但仍应在内部用它稳定选择。
