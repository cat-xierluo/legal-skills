# PDF 文书整理识别规则

本文件记录 OCR 后 PDF 的基础识别规则。AI 先用这些规则形成拆分、合并、命名判断，再写入 `organize_manifest.json`。

建议先运行 `--inspect` 获取页面级证据，再运行 `--suggest-manifest` 生成草稿。草稿只解决“可能从哪里拆”的初筛问题，主体、案由和最终文件名仍按本文件规则复核。

## 通用拆分信号

| 信号 | 说明 | 处理 |
|------|------|------|
| 新标题 | 页首出现合同、授权委托书、函、起诉状、裁定书、通知书等标题 | 通常作为新 segment 起点 |
| 页码重启 | 出现“第1页共N页”或表格页码从 `1/N` 开始 | 通常作为新 segment 起点 |
| 签署收尾 | 出现“以下无正文”、落款、具状人、日期、盖章页 | 通常作为当前 segment 结束 |
| 表格版式切换 | 从普通正文切换为法院制式表格、信息表或勾选表 | 即使标题相同，也应单独成段 |
| 函号/案号 | 出现独立函号、案号、收件法院或发文机关 | 常见于独立函件或法院文书 |

## 通用合并信号

| 信号 | 说明 | 处理 |
|------|------|------|
| 页码连续 | 多个 PDF 显示同一文书的第1页、第2页、后续页 | 合并为同一 segment |
| 标题一致 | 首页标题相同，后续文件无新标题或只是签署页/附件页 | 通常合并 |
| 续页语义 | 后一 PDF 从条款中段、事实理由中段、落款页继续 | 通常合并 |
| 文件名机械 | 文件名只有页码范围或扫描时间，无法代表独立文书 | 结合内容判断合并 |

## 常见文书识别

### 合同

识别特征：

- 标题含“合同”“协议”。
- 开头出现委托方/代理方、甲方/乙方。
- 结尾有双方签字盖章或签约时间。

命名建议：

```text
专项法律服务合同 委托方简称.pdf
委托代理合同 客户与相对方 案由.pdf
```

委托代理合同命名规则：

- 不把本所或承办律所写入文件名，除非需要区分不同律所版本。
- 优先从“就甲方与某某之间的某纠纷一案”“为委托人与某某某案”等句式中提取相对方和案由。
- 对诉讼委托合同，推荐格式是 `委托代理合同 客户与相对方 案由.pdf`。
- 示例：`委托代理合同 张家宁与杭州叠影科技有限公司 著作权权属侵权纠纷.pdf`。

### 授权委托书

识别特征：

- 标题含“授权委托书”“授權委托書”“Power Of Attorney”。
- 出现委托人、受委托人、代理权限。
- 通常为单页，但不能只按页数判断。

命名建议：

```text
授权委托书 委托人简称.pdf
```

### 律师事务所函

识别特征：

- 标题含“函”“letter”。
- 出现函号、致某法院/机构、律师事务所落款。
- 内容通常为告知已接受委托或提交材料。

命名建议：

```text
律师事务所函 案件简称.pdf
```

### 民事起诉状

识别特征：

- 标题为“民事起诉状”。
- 普通正文版通常包含原告、被告、诉讼请求、事实与理由、此致、具状人、日期。
- 要素式起诉状通常标题后有“说明”“当事人信息”，并按 `1/9`、`2/9` 等表格页码连续。

区分规则：

- 普通正文版和要素式起诉状即使案由、当事人相同，也应拆成两份。
- 普通正文版命名最后加 `普通版`。
- 要素式起诉状命名最后加 `要素式`。

命名建议：

```text
民事起诉状 当事人简称 普通版.pdf
民事起诉状 当事人简称 要素式.pdf
```

## 低置信度处理

如果标题模糊、日期疑似 OCR 错误、边界只有单一信号支持：

1. `confidence` 写 `medium` 或 `low`。
2. `needs_review` 写 `true`。
3. 文件名保守使用 `待确认 页码.pdf` 或 `疑似文书名称 页码.pdf`。
4. 不把不确定日期、案号或主体写入文件名。
