# code2patent — 代码仓库转专利交付

从已开发的代码项目中提取技术实现证据，围绕候选专利方案生成技术交底书，并以"权利要求布局卡 → 发明专利初稿"两步法继续生成接近可申报版的中国发明专利起草材料。

## 功能概览

- **代码证据提取**：读取代码仓库，识别自研模块与第三方依赖边界，提取可支持专利申请的技术证据
- **技术交底书生成**：将工程实现转换为适合研发人员与专利代理师协作的结构化交底书
- **权利要求布局**：先生成权利要求布局卡与代码证据矩阵，再据此扩展为完整初稿
- **发明专利初稿**：输出包含说明书、权利要求书草稿、摘要的接近可申报版初稿，附带自检表
- **候选方案挖掘**：如果尚无候选方案，可从代码中自动挖掘可专利的技术方案

## 适用场景

- 代码已开发完成，需要将技术实现转交给专利代理师
- 已有候选专利方案清单，需要映射到代码中的具体实现
- 希望从代码仓库中挖掘可申请专利的技术方案
- 为专利代理师准备权利要求布局和发明专利初稿

## 使用方式

本技能面向 **Claude Code** 用户。在 Claude Code 中安装后，通过对话触发：

```
/code2patent
```

### 最小输入

建议至少提供：

1. 代码仓库路径或核心模块路径
2. 候选专利方案清单
3. PRD、产品方案或需求说明（至少一种能解释项目目标的材料）
4. 每个方案对应的目标效果或待解决技术问题

### 输出层级

| 层级 | 产物 |
|------|------|
| L1 代码证据映射 | 方案-代码证据映射表 |
| L2 技术交底书 | 标准技术交底书 |
| L2.5 权利要求布局 | 布局卡 + 证据矩阵 |
| L3 发明专利初稿 | 说明书 + 权利要求书草稿 + 摘要 + 自检表 |
| L4 方案挖掘 | 候选技术方案清单 + 优先级建议 |
