# 中国专利号格式说明

中国专利有两套编号体系，**搞混会导致查不到**。本篇是 SKILL.md 的展开参考，按需加载。

## 申请号

提交专利申请时的编号，13 位数字 + 校验位。

| 示例 | 说明 |
|------|------|
| `202421964517.8` | 13 位数字 + 小数点后 1 位校验位 |

**结构（2003 年后）**：

```
2024  2  1964517  .8
└┬──┘ └┬┘ └──┬──┘ └┬┘
 年份   类型   序列号  校验位
```

- 第 1-4 位 = 申请年份
- 第 5 位类型码：1=发明，2=实用新型，3=外观设计，8=PCT 发明，9=PCT 实用新型
- 第 6-12 位 = 序列号
- 第 13 位（小数点后）= 校验位

## 公告号（公开号）

专利公开/授权后的编号，格式为 `CN + 序列号 + 种类代码`。

| 示例 | 种类 |
|------|------|
| `CN223081266U` | 实用新型授权 |
| `CN118198150A` | 发明公布 |
| `CN118198150B` | 发明授权 |

**种类代码**：

- `A` = 发明专利申请公布（公开说明书，提交时的文本）
- `B` = 发明专利授权公告（授权说明书，最终确定的权利文本）
- `U` = 实用新型专利授权公告
- `S` = 外观设计专利授权公告

## 发明 A vs B 的区别

发明专利有两次公开：

1. 先出 `A`（申请公开）——提交时的文本
2. 授权后再出 `B`（授权公告）——经审查修改后的最终版本

同一件专利的 A 和 B **内容可能不同**。诉讼、无效等需要授权版本的场景**认准 `B`**。

## 默认下载规则

发明专利优先下载 `B`（授权公告说明书），除非用户明确指定要 `A`。

## 申请号与公告号无数学对应

申请号不能直接推导出公告号。Google Patents 等平台用公告号索引，若用户只给了申请号，需先用其他平台（度衍、PatentGuru）查对应公告号。`google_patents.py` 会尝试自动匹配，但不保证 100% 成功。
