# 场景九:无效宣告抗辩

> 适用:在实用新型/外观设计/发明专利侵权诉讼中,被诉侵权方几乎必然提起专利无效宣告请求。本场景为专利权人/原告律师准备应对预案。
> 法律依据:《专利法》第 45-46 条 + 最高法专利法司法解释。

---

## 一、为什么必须重视无效宣告

### 1.1 现实数据

- **90% 以上的实用新型侵权案件**会被对方提起无效宣告请求
- 外观设计侵权案件中比例约 70-80%
- 发明专利侵权案件中比例约 30-50%
- **侵权诉讼通常会中止审理**,等待无效宣告结果(除非法院认为可以不中止)

### 1.2 战略意义

对原告(专利权人):
- 无效宣告决定直接决定侵权诉讼能否继续
- 全部权利要求被无效 → 侵权诉讼"釜底抽薪",败诉
- 部分权利要求被无效 → 重新选择权利要求主张,可能胜诉也可能败诉
- 全部权利要求维持有效 → 侵权诉讼正常推进

对被告(涉嫌侵权方):
- 无效宣告是"反守为攻"的核心武器
- 即使最终未能无效,也可拖延诉讼进程,争取谈判筹码

## 二、无效宣告的常见理由

### 2.1 实体理由

| 理由 | 法律依据 | 常见情形 |
|------|----------|----------|
| **新颖性** | 专利法 §22.2 | 权利要求技术方案被现有技术(申请日前公开)完全公开 |
| **创造性** | 专利法 §22.3 | 权利要求技术方案对**本领域普通技术人员**是显而易见的 |
| **实用性** | 专利法 §22.4 | 技术方案无法在产业上制造/使用,或无法解决声称的技术问题 |
| **充分公开** | 专利法 §26.3 | 说明书未充分公开,本领域技术人员无法实现 |
| **权利要求清楚** | 专利法 §26.4 | 权利要求不清楚、不能得到说明书支持、缺少必要技术特征 |
| **修改超范围** | 专利法 §33 | 权利要求修改超出原说明书/权利要求书记载的范围 |

### 2.2 实用新型特有风险

- 实用新型**未经实质审查**(仅做初步审查/形式审查 + 明显新颖性审查)
- 因此**新颖性、创造性**问题通常是实用新型最常见的被无效理由
- 实用新型被无效的概率显著高于发明专利(经验值约 50-70% 部分或全部无效)

### 2.3 创造性判断的"三步法"

最高法司法解释(法释〔2009〕21 号)第 22 条:创造性判断采用"**三步法**":

```
第一步:确定最接近的现有技术
第二步:确定发明的区别技术特征和实际解决的技术问题
第三步:判断要求保护的发明对本领域技术人员是否显而易见
```

实务中,创造性是最常见的无效理由(占实用新型无效案件的 60%+),也是技术含量最高、最复杂的争辩战场。

## 三、应对策略

### 3.1 总体思路

```
1. 起诉前/律师函发出前:同步做稳定性评估
2. 收到无效宣告请求后:6 个月内提交答复意见
3. 无效宣告审理中:逐项抗辩,必要时修改权利要求
4. 无效决定出来后:评估是否提起行政诉讼
```

### 3.2 时间节点

| 节点 | 时限 | 关键工作 |
|------|------|----------|
| 收到无效宣告请求 | 1 个月内 | 评估对方证据强度,制定抗辩策略 |
| 提交答复意见 | 自收到通知起 **4 个月内**(可延期 2 个月) | 撰写答复意见,提交修改文本(如有) |
| 口头审理 | 通常 6-12 个月 | 现场陈述+质证 |
| 无效决定 | 通常 9-15 个月 | 等待合议组裁定 |
| 行政诉讼(一审) | 收到决定后 6 个月内 | 若不服,起诉至北京知识产权法院 |
| 行政诉讼(二审) | 一审后 6 个月内 | 上诉至最高人民法院知识产权法庭 |

### 3.3 权利要求修改的"度"

**允许的修改**:
- 权利要求的删除
- 技术方案的删除
- 权利要求的进一步限定(将说明书/原权利要求中的技术特征加入)
- 同一权利要求中技术特征的合并

**禁止的修改**:
- 修改超出原说明书/权利要求书记载的范围
- 引入原权利要求中未要求保护的技术方案
- 扩大保护范围

**修改策略**:
- **"权利要求合并"**:将多权利要求合并成一个,缩小保护范围但提高稳定性
- **"从权并入权 1"**:将从属权利要求的附加特征并入独立权利要求,提高权 1 的新颖性/创造性
- **"删除式缩小"**:删除部分技术方案/权利要求
- 注意:修改文本是"双刃剑",可能影响在先的侵权比对结论

### 3.4 应对常见无效理由的答辩要点

**针对新颖性抗辩**:
- 论证对方引用的"现有技术"未公开权利要求的全部技术特征
- 论证技术领域、技术问题、技术效果存在实质差异
- 主张对方引用的证据是"抵触申请"而非"现有技术"(注意时点)

**针对创造性抗辩**:
- 论证区别技术特征带来"预料不到的技术效果"
- 论证本领域技术人员**没有动机**将多份现有技术结合(无结合启示)
- 论证技术方案**解决了长期未解决的技术问题**(技术偏见)
- 论证现有技术中**存在技术障碍/技术偏见**,本领域技术人员不会想到改进

**针对充分公开抗辩**:
- 论证说明书**有明确记载**,本领域技术人员能实现
- 附图、技术效果、具体实施方式都是支撑材料

**针对权利要求不清楚**:
- 论证术语在**说明书中有定义**,本领域技术人员无歧义理解
- 论证术语在**本领域有公认含义**

## 四、现有技术检索

### 4.1 检索数据库

| 数据库 | 覆盖范围 | 适用 |
|--------|----------|------|
| **CNIPA 专利公布公告系统**(epub.cnipa.gov.cn) | 中国专利 | 基础 |
| **Google Patents** | 全球专利 | 广泛 |
| **Patentics/合享智慧** | 中国专利 + 智能语义 | 高级 |
| **incoPat** | 全球专利 | 商业 |
| **百度学术/Google Scholar** | 中英文学术论文 | 补充 |
| **WIPO PatentScope** | 国际专利 | 涉外 |
| **中国知网(CNKI)** | 学位论文/期刊 | 国内非专利文献 |

### 4.2 检索方法

| 方法 | 说明 | 适用 |
|------|------|------|
| **关键词检索** | 用权利要求的技术特征关键词检索 | 初步 |
| **分类号检索** | 用 IPC/CPC 分类号检索 | 精准定位技术领域 |
| **引证链追溯** | 追溯专利的引证文献 | 找到最接近现有技术 |
| **申请人/发明人追溯** | 追溯同申请人/发明人的在先申请 | 找到抵触申请/相关技术 |
| **语义检索** | 用 Patentics/incoPat 等做语义检索 | 找到概念相关的现有技术 |

### 4.3 检索关键词构建(以"某机械连接结构"专利为例)

假设待分析专利为:某机械连接结构专利,涉及"A 部件 + B 部件 + 卡接/转动连接"的技术方案。

**核心关键词组合**:
- 连接结构 / 锁定机构 / 卡接机构
- A 部件 / B 部件 / 第一连接件 / 第二连接件
- 转动连接 / 铰接 / 卡接 / 磁吸
- 上下结构 / 双层结构 / 嵌套结构

**IPC 分类号**:
- F16B(连接件 / 紧固件)
- E05B(锁 / 钥匙 / 锁芯)

**检索式示例**:
```
(连接 AND 锁定) OR (卡接 AND 转动) OR
(A 部件 AND B 部件 AND 连接) OR
(IPC: F16B AND 双层) OR
(IPC: E05B AND 嵌套)
```

**关键提示**:
- 关键词的**穷尽性**比"精确"更重要 — 列出所有同义词、近义词、变体
- 善用**专利权人/发明人追溯** — 同主体的在先申请往往是最佳现有技术
- 善用**引证链** — 专利的"被引证"文献通常是审查员认定的相关现有技术

## 五、稳定性评估报告(原告方内部使用)

### 5.1 评估维度

| 维度 | 评分标准 | 权重 |
|------|----------|------|
| **新颖性** | 是否可能被申请日前的任一现有技术完整公开 | 高 |
| **创造性** | 是否可能由多份现有技术显而易见地组合 | 高 |
| **实用性** | 能否在产业上制造/使用 | 中 |
| **充分公开** | 说明书是否充分,本领域技术人员能否实现 | 中 |
| **权利要求清楚** | 权利要求边界是否清楚、得到支持 | 中 |
| **修改超范围** | 审查过程中是否做过修改,是否超范围 | 低 |

### 5.2 稳定性评级

| 评级 | 含义 | 应对策略 |
|------|------|----------|
| 🟢 强 | 各项评分均较高,被无效概率 < 20% | 正常推进诉讼,无需特别准备 |
| 🟠 中 | 1-2 项存在风险,被无效概率 20-60% | 准备修改预案+充分检索现有技术 |
| 🔴 弱 | 多项存在风险,被无效概率 > 60% | 重新评估是否起诉,准备权利要求合并预案 |

### 5.3 输出模板

```markdown
## 稳定性评估报告

### 专利信息
- 专利号:[填写]
- 权利要求数:[填写]
- 独立权利要求数:[填写]
- 申请日:[填写]
- 授权日:[填写]
- 审查过程中的修改:[列出]

### 各项评估

| 维度 | 评分 | 风险点 | 应对预案 |
|------|------|--------|----------|
| 新颖性 | 🟢/🟠/🔴 | [具体] | [预案] |
| 创造性 | 🟢/🟠/🔴 | [具体] | [预案] |
| 实用性 | 🟢/🟠/🔴 | [具体] | [预案] |
| 充分公开 | 🟢/🟠/🔴 | [具体] | [预案] |
| 权利要求清楚 | 🟢/🟠/🔴 | [具体] | [预案] |
| 修改超范围 | 🟢/🟠/🔴 | [具体] | [预案] |

### 现有技术检索结论
- 检索日期:[填写]
- 检索人:[填写]
- 数据库:[列出]
- 检索结果数:[N] 篇
- 最相关现有技术:
  - [现有技术 1]:[链接/编号]
  - [现有技术 2]:[链接/编号]
- 与权利要求的对比分析:
  - [现有技术 1]:[是否完整公开/区别技术特征/分析]

### 稳定性结论
- 综合评级:🟢/🟠/🔴
- 被无效概率评估:X%
- 建议:
  - [ ] 正常推进诉讼
  - [ ] 准备权利要求修改预案
  - [ ] 重新评估诉讼策略
  - [ ] 暂缓起诉,先做稳定性补强
```

## 六、原告律师的具体工作清单

### 6.1 起诉前(必须在起诉前 1-2 周完成)

- [ ] 完成稳定性评估报告
- [ ] 检索现有技术,列出最相关的 3-5 篇
- [ ] 评估被无效风险,形成"权利要求修改预案"
- [ ] 与客户沟通稳定性风险,设定合理预期
- [ ] 评估是否需要"补强专利"(如增加从属权利要求)

### 6.2 收到无效宣告请求后

- [ ] 在 1 个月内完成对方证据分析
- [ ] 在 4 个月内提交答复意见(可延期 2 个月)
- [ ] 评估修改权利要求的"度",与发明人/客户沟通
- [ ] 准备口头审理(模拟质证)
- [ ] 与客户沟通诉讼中止的可能性

### 6.3 口头审理中

- [ ] 当庭陈述+质证
- [ ] 现场回应合议组问题
- [ ] 必要时现场修改权利要求
- [ ] 与客户沟通决定作出后的应对方案

### 6.4 收到无效决定后

- [ ] 评估决定内容,判断是否全部/部分/无被无效
- [ ] 评估是否需要行政诉讼(6 个月起诉期)
- [ ] 评估对侵权诉讼的影响(是否需要中止等待)
- [ ] 与客户沟通下一步策略

## 七、关键风险提示

1. **实用新型案件必须做稳定性评估** — 不能因为"已授权"就假设专利稳定
2. **修改预案必须提前准备** — 临时修改会打乱整体策略
3. **现有技术检索要趁早** — 起诉后才发现被无效风险,已无回旋余地
4. **与客户沟通稳定性风险** — 不能让客户因专利稳定问题产生不切实际的预期
5. **行政诉讼时效** — 6 个月起诉期不能错过
6. **侵权诉讼中止问题** — 法院通常会中止等待无效决定,需提前评估
