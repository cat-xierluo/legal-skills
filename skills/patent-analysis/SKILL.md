# 专利分析工具

## 概述

本技能用于系统化分析专利文件,支持 9 种核心场景,采用**四层分析模型**(权利要求解释 → 特征比对 → 等同判断 → 抗辩与反诉),从单一特征比对扩展到全案件维度的风险预判。

## 第一部分:专利文件结构

### 1. 扉页(基本信息)

| 信息项 | 说明 |
|--------|------|
| 专利号 | 专利的唯一标识 |
| 申请日 | 保护期限起算点 |
| 专利权人 | 权利归属 |
| IPC 分类号 | 技术领域分类 |

### 2. 权利要求书(核心)

**这是专利保护范围的法律界定,是分析的核心。**

- **独立权利要求**:定义专利的**最大保护范围**,是侵权判断的基准
- **从属权利要求**:在独立权利要求基础上**增加**技术特征,保护范围更窄

**阅读技巧**:
- 独立权利要求通常以"一种..."开头
- 从属权利要求以"根据权利要求 X 所述的..."开头
- 关注"其特征在于"之后的内容

### 3. 说明书(解释权利要求)

| 章节 | 分析用途 |
|------|----------|
| 背景技术 | 理解专利要解决的问题 |
| 发明内容 | 理解核心创新点 |
| 具体实施方式 | 理解权利要求的含义 |

**重要原则**:权利要求中的术语,说明书有定义的,按说明书定义理解。

### 4. 摘要

技术概要,仅供参考,**不作为保护范围的依据**。

## 第三部分:核心方法论 — 四层分析模型

> 实际侵权判断是**四层递进**的,单层"特征比对"不足以应对实务。本模型是本 skill v2.0 的核心升级。

```
第一层:权利要求解释 → 术语含义、保护范围界定
    ↓
第二层:技术特征比对(全面覆盖原则)
    ↓ 字面侵权不成立?
第三层:等同侵权分析(三步检验法)
    ↓
第四层:抗辩与反诉预判(对方可能的攻击)
    ↓
最终结论 + 案件整体风险评估
```

### 第一层:权利要求解释

**目标**:确定权利要求 1 的保护范围,识别需要解释的术语。

**关键动作**:
1. 识别权利要求中**需要解释的术语**(非本领域通用词、有歧义的表述)
2. 按**解释规则**确定术语含义:
   - 说明书有定义 → 按说明书定义
   - 说明书无定义 → 按**本领域普通技术人员**的通常理解
   - 仍无 → 参考字典/工具书
3. 检索**专利审查文档**(意见通知书/复审决定/无效决定),看专利权人是否做过限缩性解释(影响等同原则的适用)
4. 综合上述,界定**权利要求 1 的"最大保护范围"**

**详见** [03-infringement-comparison.md §第一层](references/03-infringement-comparison.md#第一层权利要求解释)

### 第二层:技术特征比对(全面覆盖原则)

**目标**:判断涉嫌侵权产品是否字面落入权利要求 1。

**关键动作**:
1. 将权利要求 1 拆解为**最小技术特征单元**(F1、F2、...Fn)
2. 逐项比对涉嫌侵权产品,标注 **✅ 相同 / 🟡 推定 / ❌ 不同 / ❓ 待核实**
3. 全面覆盖原则:全部必要特征覆盖 → 字面侵权成立
4. 从属权利要求:作为**稳定性评估**而非侵权成立判断(详见下文)

**详见** [03-infringement-comparison.md §第二层](references/03-infringement-comparison.md#第二层技术特征比对全面覆盖原则)

### 第三层:等同侵权分析(防御性预备)

**触发条件**:第二层结论为"字面侵权不成立"时,启动第三层。

**核心方法 — 三步检验法**(Way-Function-Result Test):
- 涉嫌技术特征与权利要求特征**以基本相同的手段**
- **实现基本相同的功能**
- **达到基本相同的效果**
- 且**本领域普通技术人员无需创造性劳动**即能联想到

**限制**:
- 全部技术特征原则(逐特征判断)
- 禁止反悔原则(审查档案内限缩解释的范围不得反悔)
- 可预见性规则

**详见** [08-doctrine-of-equivalents.md](references/08-doctrine-of-equivalents.md)

### 第四层:抗辩与反诉预判

**触发条件**:无论第二/三层结论如何,启动第四层,预判对方可能的攻击。

**关键预判**:
- 对方可能的事实抗辩(不侵权、现有技术、在先使用)
- 对方可能的法律抗辩(**专利无效**、禁止反悔、滥用权利)
- 对方可能提起的反诉(确认不侵权之诉、损害赔偿之诉)

**核心战略意义 — 无效宣告应对**:
- 90% 以上实用新型案件会被对方提起**专利无效宣告请求**
- 法院通常会中止侵权诉讼,等待无效决定
- 必须在起诉前就做好**稳定性评估 + 权利要求修改预案**
- 详见 [09-invalidation-defense.md](references/09-invalidation-defense.md)

**详见** [03-infringement-comparison.md §第四层](references/03-infringement-comparison.md#第四层抗辩与反诉预判)

### 四层模型的输出

完成四层分析后,产出:
- **最终结论**:字面侵权 / 等同侵权 / 不侵权
- **案件整体风险评估**:🟢/🟠/🔴
- **后续建议**:律师函 / 起诉 / 谈判 / 撤诉
- **可视化呈现**(可选):[10-visualization.md](references/10-visualization.md) 提供特征洋葱图、侵权双栏图

## 第四部分:从属权利要求的"战略价值"

> **这是侵权判断中最容易出错的地方!**

### 核心原则

1. **侵权判断首先看独立权利要求 1**:它定义最大保护范围,是侵权判断的基准
2. **从属权利要求增强侵权稳定性**:产品落入的从属权利要求越多,侵权越稳固
3. **从属权利要求不影响侵权成立判断**:即使未落入,只要落入权 1 仍构成侵权
4. **从属权利要求是"后备阵地"**:如果权 1 被无效,可以将从属权利要求的特征并入权 1(权利要求合并),维持专利有效

### 战略意义图示

```
产品落入权 1 → 基础侵权成立
     ↓
同时落入从属权利要求?
     ↓ 是                    ↓ 否
侵权稳定性更大          侵权成立但稳定性依赖权 1
```

### 真实案例(最高人民法院知识产权法庭判决)

| 案例 | 关键事实 | 判决要点 | 战略启示 |
|------|---------|---------|---------|
| 案例 1(2019) | 产品落入权 1,未落入权 2 的"V 形结构" | 仍构成侵权 | 从属权利要求不构成侵权必要条件 |
| 案例 2(2020) | 落入权 1+权 3;被告请求无效权 1;原告将权 3 特征并入权 1 | 合并后新权 1 有效,侵权成立 | 从属权利要求是"后备阵地",可挽救专利 |
| 案例 3(2018) | 一审以"未落入从属权利要求"判决不侵权 | 最高法改判侵权 | 一审常见错误,需明确指正 |

**完整案例分析** 详见 [03-infringement-comparison.md §真实案例](references/03-infringement-comparison.md#真实案例从属权利要求的战略价值)

### 错误做法 vs 正确做法

```markdown
❌ 错误做法 1:用从属权利要求否定侵权
"专利一的驱动结构是'往复式螺旋槽',产品没有这个结构,所以不侵权。"
(错因:把从属权利要求特征当作侵权必要条件)

❌ 错误做法 2:忽略从属权利要求的战略价值
"产品落入权 1 就够了,从属权利要求不重要。"
(错因:没有评估侵权稳定性)

✅ 正确做法:
1. 先判断产品是否落入权利要求 1 → 是,基础侵权成立
2. 再判断产品落入哪些从属权利要求 → 落入权 2、权 3、权 6
3. 结论:侵权成立,且稳定性高(即使权 1 被无效,权 2/3/6 仍可维持侵权认定)
4. 同时准备"权利要求修改预案"(将权 2/3/6 特征并入权 1),应对可能的无效宣告
```

## 第六部分:常见错误

| 错误 | 正确做法 |
|------|----------|
| 只看摘要不看权利要求 | 权利要求是法律保护范围的依据 |
| 混淆独立和从属权利要求 | 先分析独立权利要求,再考虑从属权利要求 |
| 忽略技术特征的限定词 | "竖直"、"滑动"、"转动"等限定词决定保护范围 |
| 用实施例代替权利要求 | 实施例只是举例,保护范围以权利要求为准 |
| 直接给出侵权结论 | 应先进行特征分解,再逐项比对 |
| 忽略说明书的解释作用 | 说明书用于解释权利要求中的术语 |
| 把从属权利要求特征当作侵权判断条件 | 只用独立权利要求 1 的特征判断 |
| **忽略等同原则**(v2.0 新增) | 字面侵权不成立时,必须启动第三层等同分析 |
| **忽略对方无效宣告反诉**(v2.0 新增) | 90% 实用新型案件会遇到,必须做稳定性预案 |
| **跳过抗辩预判直接给结论**(v2.0 新增) | 第四层抗辩预判是法律服务方案的基础 |

## 第八部分:注意事项

1. 本技能提供的是**初步技术分析**,不构成法律意见
2. 实际侵权判定需要考虑:**等同原则、禁止反悔原则、现有技术抗辩、专利有效性**(详见第三部分四层模型)
3. 建议在重要决策前咨询专业专利律师
4. 专利状态(有效/失效/被无效)需要核实
5. 不同法域的专利法可能存在差异
6. **⚠️ 核对专利权人**:产出法律文书前,必须核实每个专利的权利人名称
   - 多个专利可能有不同的权利人(即使属于同一专利组合)
   - 权利人名称必须与专利登记信息完全一致
   - 名称错误可能导致法律文书无效
7. **⚠️ 实用新型案件 90% 会被无效**(v2.0 新增):起诉前必须做稳定性评估,准备权利要求修改预案
